Neural Network 기본 구조

핵심 직관 NN은 데이터를 보여주면서 스스로 패턴을 찾는 "함수 근사기". 입력 → 출력을 만드는 함수를 규칙 대신 데이터로 학습한다.

레이어, 가중치, 활성화함수 세 가지로 구성된다. 각 연결선에 가중치(weight)가 붙어 있으며, 학습이란 이 가중치를 조금씩 조정하는 과정이다.

그림 1. Neural Network 기본 구조
레이어입력층 → 은닉층(들) → 출력층으로 구성
가중치각 연결선에 부여된 숫자. 학습으로 조정됨
활성화함수비선형성 부여 (ReLU, Sigmoid 등). 없으면 선형 함수의 연속에 불과
학습 목표정답 레이블과의 차이(Loss)를 최소화하도록 가중치 조정
2

역전파 (Backpropagation)

핵심 직관 "틀린 정도(Loss)를 뒤에서 앞으로 전달하면서, 각 가중치가 얼마나 책임이 있는지 계산하고 조정하는 과정"

데이터셋은 세 가지로 나눈다. Train set으로 가중치를 실제로 조정하고, Validation set으로 학습 중간 성능을 확인하며(가중치 조정에는 미사용), Test set으로 최종 성능을 측정한다(딱 한 번만 사용). 이렇게 나누는 이유는 과적합(Overfitting) — 훈련 데이터만 외워버리는 현상 — 을 방지하기 위해서다.

그림 2. 순전파 → Loss 계산 → 역전파 흐름
학습률 (Learning Rate) 가중치를 얼마나 크게 조정할지 결정하는 값. 너무 크면 정답을 지나치고, 너무 작으면 학습이 느리다. 이를 제어하는 알고리즘이 SGD, Adam 같은 옵티마이저(Optimizer).
3

Transformer & Attention

핵심 직관 모든 토큰을 한 번에 보면서 서로의 관계를 동시에 계산한다. Attention은 결국 Q, K를 활용해 V를 반환하는 함수.

Attention 공식

Attention(Q, K, V) = softmax(Q·Kᵀ / √d) · V

Q, K의 내적으로 유사도 점수를 구하고, √d로 나눠 스케일링한 뒤, Softmax로 확률값(가중치)으로 변환한다. 이 가중치에 V를 곱해 최종 출력 벡터를 얻는다.

그림 3. Attention 메커니즘 — "it"이 "cup"에 집중하는 과정
Context windowAttention이 실제로 계산되는 범위 (예: GPT-4는 최대 128,000 토큰)
Quadratic scalingN개 토큰 → N² 쌍 계산. Context 길어질수록 비용이 제곱으로 증가
Softmax점수 차이를 지수함수로 증폭 → Winner를 두드러지게 만듦
Multi-head Attention여러 Attention head가 병렬로 다양한 관점(문법적·의미적)에서 계산
Transformer 구조[Attention → Feed Forward] × N층. 층이 깊을수록 추상적 표현 학습
4

Vision Encoder (ViT)

핵심 아이디어 이미지를 16×16 픽셀 패치로 쪼개 토큰처럼 취급하면, Transformer를 그대로 재사용할 수 있다.

Vision Encoder는 역할 이름(이미지 → 벡터 변환 모듈)이고, ViT는 그 역할을 Transformer로 구현한 방식이다. CNN(ResNet 등)이 Vision Encoder를 담당하던 시대(~2020)에서, ViT 등장 이후 최신 VLM들은 대부분 ViT를 Vision Encoder로 사용한다.

그림 4. ViT — 이미지 패치를 토큰처럼 처리하는 구조
5

LLM — 다음 토큰 예측

핵심 GPT 계열 LLM은 Encoder 없이 Decoder only 구조로, 입력 이해와 텍스트 생성을 하나의 Decoder가 모두 담당한다.

세 가지 Transformer 구조

Encoder only

양방향 Self-Attention. 입력 전체를 한 번에 봄.

용도: 이해·분류·임베딩. 생성 불가.

예: BERT, ViT

Decoder only ⭐ LLM 주류

단방향 Masked Self-Attention. 앞 토큰만 봄.

입력 이해 + 생성 모두 Decoder가 담당.

예: GPT, LLaMA, Claude, RT-2 기반

Encoder-Decoder

Encoder가 입력 이해(양방향), Decoder가 생성(단방향).

입력·출력 언어가 명확히 분리될 때 유리.

예: T5, 번역 모델
그림 5. 세 가지 Transformer 구조 비교 & Autoregressive 생성
6

RNN vs Transformer

딥러닝과의 관계 Transformer는 딥러닝의 한 종류다. RNN, CNN도 마찬가지. 구조가 다를 뿐 모두 역전파로 학습하는 NN이다.

본질적 차이는 시퀀스를 어떻게 처리하느냐다.

처리 방식RNN: 단어를 하나씩 순서대로 | Transformer: 모든 토큰을 동시에
정보 전달RNN: hidden state 순차 전달 → 뒤로 갈수록 희석 | Transformer: Attention으로 모든 토큰 직접 참조
병렬 처리RNN: 불가 (순서 의존) | Transformer: 가능 → GPU 활용 극대화
장거리 의존성RNN: 약함 | Transformer: 강함 (거리 무관하게 직접 참조)
RNN: The → cat → sat → on h1 → h2 → h3 → h4 (순차, 앞 정보 희석) Transformer: The cat sat on ↕ ↕ ↕ ↕ (동시, 모든 쌍 직접 참조)
Transformer가 주류가 된 핵심 이유 병렬화 가능 → GPU 활용 극대화 → 대규모 학습 가능. RNN은 순서 의존성 때문에 병렬화가 근본적으로 어렵다.
7

Pre-training vs Fine-tuning

비유 의대생이 6년간 의학 전반을 공부하는 게 pre-training, 졸업 후 흉부외과 전문의 수련을 받는 게 fine-tuning.
데이터Pre-training: 수백억 토큰 (인터넷 전체 규모) | Fine-tuning: 수만~수백만 건 (특정 도메인)
목표Pre-training: 범용 언어 이해 획득 | Fine-tuning: 특정 태스크 특화
비용Pre-training: 수개월 / 수천억 원 | Fine-tuning: 상대적으로 저렴
결과Pre-training: Base Model | Fine-tuning: 특화 모델
Pre-training (범용 학습) ↓ Base Model ↙ ↘ Fine-tuning A Fine-tuning B 의료 데이터 추가 로봇 액션 데이터 추가 → 의료 전문 모델 → VLA 모델 (RT-2)
RT-2 맥락 PaLM-E 같은 대형 VLM을 pre-trained base로 가져와서, 로봇 액션 데이터로 fine-tuning한 게 RT-2다. Fine-tuning의 변형으로 LoRA 같은 기법도 있다 — 전체 가중치 대신 일부 레이어만 업데이트해서 비용을 줄이는 방법.

Parameter란 무엇인가

Parameter = Weight, 사실상 같은 말 Parameter는 모델이 학습으로 조정하는 모든 숫자. Weight는 연결선의 강도, Bias는 각 노드의 기본값 오프셋. Weight + Bias = Parameter. 실전에서는 거의 혼용한다.

왜 70억 개나 필요한가

Transformer 레이어 하나 안에 있는 행렬들의 크기를 보면 이해가 된다.

레이어 하나 (hidden dim = 4096 기준): Q 가중치 행렬: 4096 × 4096 = 16,777,216개 K 가중치 행렬: 4096 × 4096 = 16,777,216개 V 가중치 행렬: 4096 × 4096 = 16,777,216개 FFN 레이어: 훨씬 더 큼 ────────────────────────────────────────── 레이어 하나만으로 이미 수천만 개 이게 32층 쌓이면 → 수십억 개 (7B)
SmolVLM (2B)약 4GB 메모리 (FP16 기준)
LLaMA (7B)약 14GB 메모리 (FP16 기준)
GPT-4 (추정)~1T 파라미터 / 수TB 메모리
On-device 배포와의 연결 파라미터 수 = 메모리 요구량. 이것이 로봇 온디바이스 배포에서 양자화(Quantization)가 필요한 직접적인 이유다.
FP32 (32bit): 파라미터 1개 = 4 bytes FP16 (16bit): 파라미터 1개 = 2 bytes INT8 (8bit): 파라미터 1개 = 1 byte ← 양자화 INT4 (4bit): 파라미터 1개 = 0.5 bytes ← 더 공격적인 양자화 7B 모델 FP32: 28GB → INT4: 3.5GB (로봇 온디바이스 가능)

전체 요약 & RT-2 연결고리

NN가중치를 조정하는 함수 근사기. 역전파로 Loss를 최소화하며 학습
TransformerAttention(Q, K, V) 함수를 N층 쌓은 구조. 모든 토큰 관계를 동시 계산
RNN vs Transformer순차 처리 vs 병렬 처리. 병렬화 가능성이 Transformer 주류화의 핵심
ViT이미지를 패치 토큰으로 쪼개 Transformer에 입력. Vision Encoder의 현재 주류
LLMDecoder only 구조. 이전 토큰들을 보며 다음 토큰을 자기회귀적으로 생성
Pre-training수백억 토큰으로 범용 언어 이해 학습. 비용 매우 큼
Fine-tuningPre-trained 모델을 특정 태스크 데이터로 추가 학습. RT-2도 이 방식
Parameter모델의 모든 학습 가능한 숫자 (≈ Weight). 많을수록 복잡한 패턴 표현 가능

RT-2에서의 조합

텍스트 토큰: "컵을 집어라" ─┐ 이미지 패치: 카메라 영상 ─┤→ 같은 Transformer에 함께 입력 액션 토큰: 관절 각도 값 ─┘ ViT (Encoder) → 이미지 이해 LLM (Decoder only) → 액션 생성 "이미지를 읽고 → 액션을 쓴다"
핵심 통찰 ViT가 이미지를 텍스트처럼 만들어주기 때문에, 텍스트·이미지·로봇 액션 모두 Transformer 입장에선 똑같은 토큰이다. 이것이 VLM → VLA로 이어지는 핵심 아이디어.
