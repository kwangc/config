# VLA에서의 멀티모달 융합

멀티모달 융합(multimodal fusion)은 비전 토큰, 언어 토큰, 로봇 상태를 백본이 추론할 수 있는 통합된 표현으로 결합하는 과정입니다. 융합 전략의 선택은 모델 표현력, 파라미터 수, 학습 난이도, 추론 레이턴시에 영향을 미칩니다.

## 융합 문제

ViT의 비전 토큰은 학습된 시각적 임베딩 공간에 있습니다. LLM의 언어 토큰은 언어 임베딩 공간에 있습니다. 이 두 공간은 서로 다른 데이터 분포에서 독립적으로 학습되어 자연스럽게 정렬되지 않습니다. 좋은 액션을 생성하려면 모델이 다음을 해야 합니다:

1. 언어 지시사항이 주어졌을 때 어떤 시각적 영역이 관련 있는지 파악 (예: "빨간 컵을 집어라" → 빨간 컵에 어텐션)
2. 현재 상태(proprio)와 지시사항의 목표 사이의 관계 이해
3. 두 모달리티에 동시에 기반한 액션 생성

세 가지 고수준 융합 전략:

| 전략 | 융합 시점 | 설명 |
|------|----------|------|
| 조기 융합(Early fusion) | 백본 이전 | 모달리티들이 입력에서 단일 토큰 시퀀스로 결합 |
| 중간 융합(Mid fusion) | 백본 내부 | 중간 레이어에서 모달리티 스트림 간 크로스 어텐션 |
| 후기 융합(Late fusion) | 백본 이후 | 모달리티를 별도로 처리하고 액션 헤드에서만 병합 |

현대 VLA는 거의 독점적으로 조기 융합(투영을 통한 연결) 또는 중간 융합(크로스 어텐션)을 사용합니다. 후기 융합은 너무 일찍 크로스 모달 상호작용을 버립니다.

## 투영 레이어 (LLaVA 스타일)

가장 간단하고 널리 사용되는 방법: 학습된 선형 레이어 또는 소형 MLP를 사용해 시각 토큰을 LLM의 임베딩 공간으로 투영한 후 언어 토큰과 연결합니다.

```
비전 토큰: N × d_vision    (ViT에서)
투영 W: d_vision → d_llm  (학습된 선형 또는 2레이어 MLP)
투영된 토큰: N × d_llm

LLM 입력: [투영된 비전 토큰 | 언어 토큰]
```

**왜 작동하는가.** 투영 레이어가 유일한 새 구성요소입니다. ViT 백본과 LLM 백본은 동결 상태를 유지할 수 있으며, 투영 가중치만 학습하면 됩니다(액션 헤드와 함께). 이로 인해 파라미터 효율이 매우 높고 학습이 쉽습니다.

**사용처:** LLaVA, RT-2, OpenVLA, 대부분의 오픈소스 VLA.

**한계.** 투영은 고정 선형 맵입니다 — 언어 지시사항에 의해 조건화되지 않습니다. LLM은 자기 어텐션을 통해 내부적으로 모든 크로스 모달 추론을 해야 합니다. 이것이 효과적이려면 큰 LLM이 필요합니다.

## 크로스 어텐션 융합

크로스 어텐션(cross-attention)은 언어 표현이 시각 표현에 직접 어텐션을 수행할 수 있게 합니다(또는 그 반대):

```
Flamingo 스타일:
  언어 은닉 상태 → 쿼리 Q
  시각 패치 토큰 → 키 K, 값 V
  fused = softmax(Q K^T / sqrt(d)) V

동결된 LLM 자기 어텐션 레이어 사이에 크로스 어텐션 레이어 삽입
```

이것은 투영 레이어보다 더 표현력이 있습니다: 각 언어 토큰이 필요한 시각 정보를 선택적으로 추출할 수 있습니다. 크로스 어텐션 레이어는 파라미터와 학습 복잡성을 추가하지만, 시각 정보가 언어 쿼리에 의해 조절될 수 있습니다 — 다른 지시사항이 이미지의 다른 영역에 어텐션해야 할 때 유용합니다.

**사용처:** Flamingo, IDEFICS, Otter.

**트레이드오프.** 더 많은 파라미터, 더 많은 하이퍼파라미터 (크로스 어텐션 레이어 수, 삽입 위치). 동결 LLM + 크로스 어텐션 패턴은 두 모달리티의 표현을 저하시키지 않도록 신중한 최적화가 필요합니다.

## 연결 + 인과 트랜스포머

최근 VLA에서 가장 일반적인 방법: 비전 토큰을 언어 토큰 시퀀스 앞에 단순히 붙이고, 인과 트랜스포머가 혼합 시퀀스에 어텐션을 수행하게 합니다.

```
입력 시퀀스 = [vis_1, ..., vis_N, lang_1, ..., lang_L, state_1, action_1, ...]

인과 어텐션 마스크: 각 토큰이 이전의 모든 토큰에 어텐션
```

이것은 가장 간단한 구현이며 백본 트랜스포머에 아키텍처 변경이 필요하지 않습니다. LLM은 시퀀스 시작 부분의 텍스트 토큰처럼 비전 토큰을 처리합니다.

**장점:**
- 비전 투영 외에 추가 파라미터 없음
- 기존 LLM 인프라(flash attention, vLLM 등)와 완전히 호환
- 모든 레이어에서 시각적 + 언어적 전체 인과 맥락 사용 가능

**한계.** 인과 마스크는 언어 토큰이 시퀀스에서 이후에 오는 비전 토큰에 어텐션할 수 없습니다. 표준 관행은 비전 토큰을 먼저 배치하고 언어 토큰을 두 번째로 배치하는 것입니다 — 언어 토큰이 항상 시각적 맥락에 접근할 수 있도록.

## Q-Former (BLIP-2 스타일)

Q-Former(Query-Former)는 이미지 특성 맵에 크로스 어텐션하여 고정 길이 시각 요약을 생성하는 학습된 쿼리 토큰 집합을 도입합니다:

```
학습 가능한 쿼리: K × d  (K << N, 예: K=32, N=196)
이미지 특성: N × d_vision
Q-Former 크로스 어텐션: 쿼리가 이미지 특성에 어텐션
출력: K 요약 토큰  (이미지 해상도와 무관하게 동일한 K)
```

**핵심 특성:** 출력 크기가 이미지 해상도나 패치 크기와 무관하게 K 토큰으로 고정됩니다. 이를 통해 LLM의 컨텍스트 창을 초과하지 않고 매우 고해상도 이미지나 많은 카메라를 사용할 수 있습니다.

**트레이드오프.** K 쿼리 토큰은 병목입니다: 압축에서 손실된 정보는 이후에 복구할 수 없습니다. K가 너무 작으면 세밀한 공간 세부 정보가 손실됩니다.

**사용처:** BLIP-2, InstructBLIP, 고해상도 입력을 사용하는 일부 로봇 조작 연구.

## 로봇 상태에 대한 조건화

고유감각 상태(proprioceptive state)도 백본의 맥락에 융합되어야 합니다.

### 토큰으로 선두에 추가

```
state_emb = MLP(proprio)  → 형태: (d,)
토큰 시퀀스에 선두로 추가: [state_token | vis_tokens | lang_tokens]
```

백본이 입력 맥락의 일부로 상태 토큰에 어텐션합니다. 이것은 가장 표현력 있는 방법이며 현대 VLA에서 가장 일반적입니다 (π0, ACT). 상태 토큰은 모든 후속 토큰이 어텐션할 수 있도록 첫 번째에 배치됩니다.

### CLS 임베딩에 추가

```
cls_emb = cls_emb + MLP(proprio)
```

간단하며 시퀀스에 토큰을 추가하지 않습니다. 액션 헤드의 백본 읽기값으로 CLS 토큰이 사용될 때 작동합니다. 덜 표현력 있습니다 — proprio 정보가 강한 시각적, 언어적 특성 속에 섞입니다.

### 인터리브된 상태 토큰

순환 또는 이력 조건화 아키텍처의 경우: 다중 타임스텝 관찰 창의 각 타임스텝에 상태 토큰을 포함합니다. 백본이 상태가 시간에 따라 어떻게 변했는지 추론할 수 있게 합니다. 더 비싸지만 진행 추적이 필요한 태스크에 필요합니다.

## 트레이드오프 요약

| 융합 방법 | 표현력 | 파라미터 오버헤드 | 학습 복잡성 | 일반적 사용처 |
|----------|-------|-----------------|------------|-------------|
| 투영 (LLaVA) | 중간 | 매우 낮음 (선형 레이어 하나) | 낮음 | RT-2, OpenVLA, 대부분의 오픈소스 VLA |
| 크로스 어텐션 | 높음 | 중간 (새로운 어텐션 레이어) | 중간 | Flamingo 스타일; VLA에서 덜 일반적 |
| 연결 + 인과 | 중간-높음 | 없음 | 매우 낮음 | π0, 현대 대부분의 VLA |
| Q-Former | 높음 (압축 포함) | 중간 (쿼리 토큰) | 중간 | 고해상도 또는 멀티카메라 시나리오 |

## 실용적 권장 사항

- **투영 + 연결로 시작하세요.** 잘 작동하고, 백본 변경이 최소화되며, 기존 학습 프레임워크에서 잘 지원됩니다.
- **컨텍스트 길이가 제약인 경우 Q-Former를 사용하세요.** 4개 이상의 카메라가 있거나 448×448+ 해상도를 사용하려면 Q-Former가 시각 토큰을 LLM에 도달하기 전에 압축할 수 있습니다.
- **크로스 어텐션은 표준 조작 태스크에 거의 필요하지 않습니다.** 표현력 향상은 실재하지만 일반적으로 병목이 아닙니다; 데이터 다양성과 액션 헤드 설계가 더 중요합니다.
- **항상 상태를 CLS에 합산하지 말고 토큰으로 선두에 추가하세요.** 더 표현력 있고 시퀀스에 토큰 하나만 추가됩니다.

## 생각해볼 거리

- VLA 정책이 시각적으로 유사하지만 언어 지시사항이 다른 장면에서 다르게 수행된다면, 투영 레이어가 충분한 크로스 모달 정렬을 하지 못하는 것입니다 — 크로스 어텐션 레이어를 고려하거나 비전 토큰이 언어 관련 이미지 영역을 보고 있는지 확인하세요.
- 카메라를 더 추가했는데도 성능이 저하된다면 (더 많은 정보를 제공함에도), 결합된 토큰 시퀀스가 백본이 효과적으로 어텐션하기에 너무 길어졌을 가능성이 높습니다 — Q-Former 압축이나 패치 크기 증가가 정보 과부하가 문제가 되기 전에 예산을 회복시킵니다.
- 파인튜닝 중 ViT와 LLM을 모두 동결하고 투영과 액션 헤드만 학습하면 인-분포(in-distribution) 태스크에서 빠른 수렴을 볼 것이지만 새로운 물체 외양에 대한 일반화가 나쁩니다 — 매우 작은 학습률로 ViT의 마지막 두 레이어를 부분적으로 동결 해제하는 것이 종종 올바른 균형입니다.
- 로봇 상태 조건화가 현재 팔 구성을 무시하는 액션을 생성하고 있다면, 상태가 토큰으로 선두에 추가되는 대신 합산(summed)되고 있는 것입니다 — 합산은 강한 시각적, 언어적 특성 앞에서 상태의 독립적 신호를 잃게 합니다.
