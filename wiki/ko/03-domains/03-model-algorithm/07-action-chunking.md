# 액션 청킹(Action Chunking)과 시간적 일관성

액션 청킹(action chunking)은 단일 모델 호출로 여러 미래 액션을 예측하여 로봇 정책의 시간적 일관성을 개선하는 기법입니다. 단일 스텝 모방 학습 정책을 괴롭히는 오차 누적(compounding error)의 근본적 문제를 해결합니다.

## 오차 누적 문제

표준 단일 스텝 정책은 매 타임스텝마다 쿼리됩니다:

```
for each t:
  a_t = policy(obs_t)
  execute a_t
  observe obs_{t+1}
```

문제: 정책은 상태 궤적이 인간 시연자의 자연스러운 경로를 따르는 시연 데이터로 학습되었습니다. 정책이 스텝 t에서 작은 오차를 만들자마자, 결과 상태 obs_{t+1}이 학습 데이터에 있는 것과 약간 다릅니다. 정책은 이제 약간 분포 외(out-of-distribution) 상태에 있습니다. 또 다른 작은 오차가 만들어집니다. 상태가 더 멀리 벗어납니다. 충분한 스텝 후 로봇은 학습 분포에서 멀리 떨어진 상태에 있고 정책은 잘못된 액션을 생성합니다.

**이것은 모델 품질 문제가 아닙니다 — 단일 스텝 행동 복제의 구조적 특성입니다.** 완벽한 정책(학습 손실 = 0)조차도 배포 시 오차를 누적합니다. 시연이 실행 중 로봇이 방문할 수 있는 모든 가능한 상태를 커버하지 않았기 때문입니다.

심각성은 다음과 함께 증가합니다:
- 태스크 길이 (더 많은 스텝 = 더 많은 이탈 기회)
- 태스크 정밀도 (세밀한 조작에서 작은 위치 오차가 더 중요)
- 관찰 노이즈 (카메라 떨림이나 고유감각 지연이 상태 추정 오차를 증폭)

## 액션 청킹

액션 청킹은 한 번에 K개의 미래 액션 시퀀스를 예측합니다:

```
chunk = policy(obs_t) = [a_t, a_{t+1}, a_{t+2}, ..., a_{t+K-1}]
a_t, a_{t+1}, ..., a_{t+K-1} 실행   (K 스텝 동안 오픈루프)
그 후 재쿼리: chunk = policy(obs_{t+K})
```

정책은 K 스텝마다 한 번만 호출됩니다. 청크 내에서는 실행이 오픈루프입니다(새로운 관찰이 사용되지 않음).

**왜 도움이 되는가:**

1. **청크 내 시간적 일관성.** K 스텝을 커버하는 단일 예측은 내부적으로 일관성이 있어야 합니다 — 예측된 a_{t+1}은 a_t를 따르도록 설계됩니다. 이것은 K개의 독립적인 단일 스텝 예측보다 더 부드러운 궤적을 생성합니다.

2. **줄어든 쿼리 빈도.** 정책을 K배 적게 호출하면 분포 외 상태를 방문하는 속도가 줄어듭니다. 오차가 모든 스텝이 아닌 청크 경계에서만 누적됩니다.

3. **더 나은 다중 모달 액션 처리.** 전체 궤적(청크)을 예측하면 모델이 해당 기간 동안 분포의 한 모드에 전념할 수 있습니다.

4. **분할 상환된 연산 비용.** 정책이 추론이 느린 LLM(호출당 100ms)이라면, K=10 액션을 호출당 실행하면 10 Hz 제어가 1 Hz 정책 호출에 10 Hz 실행이 됩니다.

## 청크 크기 트레이드오프

| K (청크 크기) | 쿼리 빈도 | 오픈루프 기간 | 반응 능력 | 오차 누적 |
|------------|----------|------------|---------|---------|
| 1 (청킹 없음) | 매 스텝 | 0 | 완전 | 높음 |
| 5-10 | 5-10 스텝마다 | 50Hz에서 0.1-0.2s | 좋음 | 중간 |
| 20-50 | 20-50 스텝마다 | 50Hz에서 0.4-1s | 보통 | 낮음 |
| 100+ | 100+ 스텝마다 | 50Hz에서 2s+ | 나쁨 | 매우 낮음 |

**너무 작음 (K=1-3):** 사실상 여전히 단일 스텝. 오차 누적이 지속됩니다.

**너무 큼 (K=100+):** 청크가 오픈루프로 몇 초를 커버합니다. 환경이 변하면(물체가 떨어지거나, 사람이 무언가를 움직이면), 정책이 반응할 수 없습니다.

**실용적 범위:** 50 Hz 제어에서 K=10-50 (0.2-1초 예측)이 테이블탑 조작에서 잘 작동합니다. ACT는 ~30 Hz에서 K=100을 사용합니다.

## 시간적 앙상블(Temporal Ensemble) — 겹치는 청크

시간적 앙상블은 매 스텝마다 정책을 쿼리하지만 겹치는 청크 예측의 슬라이딩 버퍼를 사용하는 개선입니다:

```
매 스텝 t에서:
  정책 쿼리: chunk_t = [a_t, a_{t+1}, ..., a_{t+K-1}]
  buffer[t:t+K] += chunk_t  (가중치로 누적)

스텝 t를 커버하는 모든 예측의 가중 평균을 실행:
  action_t = sum_k w_k * chunk_{t-k}[k]   for k in 0..K-1
```

**지수 가중.** 더 최근 예측에 더 높은 가중치를 부여합니다:

```
w_k = exp(-m * k)   여기서 k는 예측의 "나이" (스텝 전)
```

현재 스텝에서 예측된 청크는 가중치 1을 받습니다. K-1 스텝 전에 예측된 청크는 가중치 exp(-m*(K-1))을 받으며, 이는 작습니다.

**왜 작동하는가.** K개의 겹치는 예측의 지수 가중 평균은 다음과 같은 궤적을 제공합니다:
- 가장 최근 예측에 의해 지배됨 (항상 현재 관찰을 반영)
- 이전 예측에서 스무딩된 정보를 전달함 (각 스텝에서 새로운 모드로 급격히 전환하지 않음)
- 개별 예측에 노이즈가 있어도 물리적으로 부드러운 관절/EEF 궤적 생성

**비용:** 정책이 매 스텝마다 호출됩니다 (K 스텝에 걸쳐 분할 상환 안 됨). 이는 추론이 제어 빈도에 충분히 빠라야 함을 의미합니다.

**사용처:** ACT (K=100 청킹과 시간적 앙상블 모두).

## 순환 아키텍처 (Recurrent Architecture)

시간적 의존성 처리를 위한 청킹의 대안: 타임스텝에 걸쳐 히든 상태를 유지하는 순환 네트워크(LSTM, GRU) 사용.

```
h_0 = 0
for each t:
  a_t, h_{t+1} = policy(obs_t, h_t)
  execute a_t
```

히든 상태 h는 이전 관찰과 액션의 정보를 전달하여, 입력에 명시적 이력 없이도 정책이 작업 진행 상황을 추적할 수 있게 합니다.

**청킹과의 비교:**

| 특성 | 청킹 | 순환 |
|-----|------|------|
| 시간적 맥락 | 오픈루프 K 스텝 예측 | 무제한 (증가하는 히든 상태) |
| 돌발 상황 반응 | 청크 경계에서만 | 매 스텝 |
| 아키텍처 변경 | 예측 헤드 확장 | 순환 백본 필요 |
| 학습 복잡성 | 표준 BC 손실 | 순차적 학습, BPTT 필요 |
| 추론 복잡성 | K 스텝마다 전체 순방향 패스 | 스텝당 작은 히든 상태 업데이트 |
| 현대 VLA에서 일반적 | 예 (지배적 방법) | 덜 일반적 |

순환 아키텍처는 현대 LLM 백본 VLA에서 덜 일반적입니다 — 트랜스포머 기반 모델은 자연스럽게 순환이 아니기 때문입니다. 반면 청킹은 출력 헤드를 확장하여 동결된 백본으로도 작동합니다.

## 하이브리드: 청킹 + 시간적 앙상블

ACT와 관련 시스템에서 사용되는 조합:

1. **청크 예측:** 정책 호출당 K개 액션을 한 번에 예측
2. **시간적 앙상블:** 매 스텝마다 쿼리하지만 K개의 겹치는 예측을 블렌딩

이것으로 다음을 얻습니다:
- 청킹의 시간적 일관성 이점 (각 호출이 내부적으로 일관된 궤적을 생성)
- 앙상블의 부드러움과 반응성 이점 (실행된 액션이 항상 최근 예측의 블렌드)

**연산 비용:** 스텝당 K× 추론 예산 (정책이 매 스텝마다 호출됨), 하지만 각 호출이 K개 액션을 생성합니다. K=100이고 추론이 20ms라면, 이는 20ms 레이턴시로 제어 빈도에서 정책을 실행해야 함을 의미합니다 — 전용 GPU에서 가능합니다.

## 요약: 제어 전략

| 전략 | 초당 모델 호출 (50 Hz에서) | 관찰 변화 반응 | 부드러움 |
|------|--------------------------|--------------|---------|
| 단일 스텝 BC | 50 | 매 스텝 | 낮음 (떨림) |
| 청킹만 (K=50) | 1 | 50 스텝마다 (1s) | 중간 |
| 시간적 앙상블만 | 50 | 매 스텝 | 높음 |
| 청킹 + 앙상블 | 50 | 매 스텝 | 높음 |

## 생각해볼 거리

- 단일 스텝 정책이 짧은 태스크(5초 미만)에서는 성공하지만 긴 태스크에서 일관되게 실패한다면, 오차 누적을 실제로 경험하고 있는 것입니다 — 이것이 더 많은 데이터가 아니라 액션 청킹이 도움이 될 것이라는 주요 신호입니다.
- 시간적 앙상블을 추가했더니 로봇 궤적이 더 부드러워지지만 태스크 경계에서 가끔 "고착"된다면 (예: 집은 후 놓기 전에 망설임), 지수 감쇠 가중치 m이 너무 낮습니다 — 오래된 청크 예측이 블렌딩된 액션을 현재 올바른 동작에서 끌어당기고 있습니다.
- 청크 크기 K가 한 제어 빈도에 맞게 조정되었는데 로봇의 제어 루프 속도를 변경하면, K를 재조정해야 합니다 (또는 청크 기간이 비례하여 변합니다) — 50 Hz에서 1초 예측을 생성하는 K는 10 Hz에서 0.2초 예측을 생성하며 너무 짧을 수 있습니다.
- 양팔 태스크에서 두 팔이 동시에 물체에 접촉해야 한다면, 단일 스텝 예측은 이것을 거의 달성하지 못합니다 — 전체 접촉 시퀀스에 걸친 액션 청킹(접근 + 접촉을 커버할 만큼 큰 K)이 모델이 하나의 일관된 청크로 동시 접촉을 계획할 수 있게 합니다.
