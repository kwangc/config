# 정책 아키텍처 패턴

이 페이지는 주요 VLA 정책 아키텍처를 나란히 비교합니다: 백본 선택, 액션 헤드, 학습 데이터 전략, 핵심 트레이드오프. 아키텍처를 선택하거나 각 시스템이 무엇을 위해 구축되었는지 이해하려 할 때 이 페이지를 사용하세요.

## RT-2 (Robotics Transformer 2)

**핵심 아이디어:** 로봇 액션을 텍스트 토큰으로 취급합니다. 대형 비전-언어 모델을 파인튜닝하여 텍스트를 예측하는 것과 동일한 방식으로 액션 토큰 시퀀스를 예측합니다.

**아키텍처:**
- 백본: PaLM-E (562B 파라미터) 또는 PaLI (55B 파라미터) VLM
- 비전 인코더: ViT 기반 이미지 인코더
- 액션 표현: 텍스트 출력에 추가된 이산화된 빈 토큰
- 디코딩: 표준 자기회귀 토큰 예측 (텍스트 생성과 동일)

**학습:**
- 공동 학습: 로봇 시연 데이터와 인터넷 규모의 VQA, 캡셔닝, 지시 따르기 데이터 혼합
- 로봇 데이터 비율: 소량 (~5%)
- 로봇 데이터셋: Google 독점 데이터; 다양한 조작 태스크
- 손실: 모든 토큰에 대한 교차 엔트로피 (언어 + 액션 토큰 동일하게 처리)

**핵심 인사이트:** 액션 토큰이 텍스트 토큰과 동일한 메커니즘으로 예측되기 때문에, 모델이 자신의 언어 이해를 활용하여 로봇 데이터에서 본 적 없는 새로운 지시사항으로 일반화할 수 있습니다. 모델이 인터넷 텍스트에서 "컵을 마커 왼쪽에 놓아라"를 이해한다면, 이 지시사항에 대한 직접적인 학습 데이터 없이도 로봇에서 실행할 수 있습니다.

**추론:**
- 자기회귀 토큰 디코딩: 순방향 패스당 하나의 토큰
- 7개 액션 차원 × 토큰당 1개 = 액션 스텝당 LLM 순방향 패스 7회
- 7B-562B 규모에서: 하드웨어에 따라 액션 스텝당 200ms-2s

**단점:**
- 순차적 토큰 생성으로 인한 느린 디코딩
- 빈 수에 따른 제한된 정밀도
- 매우 큰 모델 (562B)이 온로봇 배포에 실용적이지 않음

## OpenVLA

**핵심 아이디어:** RT-2의 오픈소스 재현 및 개선으로, 공개적으로 사용 가능한 데이터로 학습하고 가중치를 릴리즈합니다.

**아키텍처:**
- 백본: Llama-2 7B (언어) + SigLIP ViT (비전)
- 융합: LLaVA 스타일의 선형 투영을 통해 시각 토큰을 LLM 임베딩 공간으로 투영
- 액션 표현: 이산화된 빈 토큰 (RT-2와 동일)
- 디코딩: 자기회귀

**학습:**
- 데이터셋: Open X-Embodiment (OXE) — 공개적으로 사용 가능한 크로스 로봇 데이터셋 (~22가지 로봇 유형, 1M+ 궤적)
- LLaVA 스타일 VLM 체크포인트에서 파인튜닝
- 파라미터 효율적 파인튜닝을 위한 LoRA 어댑터

**핵심 인사이트:** 7B 규모가 OXE 벤치마크에서 강한 언어 일반화와 조작 성능에 충분합니다. 공개 가중치는 이전에 독점 모델 뒤에 있던 커뮤니티 연구를 가능하게 합니다.

**추론:** RT-2와 동일한 순차적 디코딩; 7B 모델은 4비트 양자화로 단일 소비자 GPU (RTX 3090/4090)에서 액션 스텝당 ~100ms로 실행 가능합니다.

**단점:**
- RT-2와 동일한 정밀도 및 속도 제한
- 액션 토크나이제이션이 다중 모달 분포를 포착하지 못함
- 세밀한 태스크에서 디퓨전 기반 방법보다 낮은 조작 정밀도

## ACT (Action Chunking with Transformers)

**핵심 아이디어:** CVAE + 트랜스포머 아키텍처를 사용해 한 번에 100개의 미래 액션 청크를 예측합니다. LLM 백본 불필요.

**아키텍처:**
- 관찰 인코더: 각 카메라에 대한 ResNet-18 또는 ViT
- 언어 조건화: 간단한 태스크 임베딩 (BERT 또는 고정 인코딩); 전체 LLM 아님
- 백본: 소형 트랜스포머 (6 인코더 + 6 디코더 레이어, d=256)
- 액션 헤드: CVAE (Conditional Variational Autoencoder)
  - 인코더: 전체 액션 청크를 받아 잠재 z 생성
  - 디코더: 트랜스포머가 z + 관찰로부터 K=100 액션 청크 디코딩
- 청크 크기: 50Hz에서 K=100 (호출당 2초 액션)
- 시간적 앙상블: 예 (K개의 겹치는 예측이 지수 가중치로 블렌딩)

**학습:**
- 데이터셋: 소량의 고품질 인간 teleop 시연 (태스크당 50-200개)
- 손실: 액션 청크에 대한 MSE + CVAE KL 발산 항
- 학습 규모: 단일 GPU에서 수 시간; 대규모 사전 학습 불필요

**핵심 인사이트:** CVAE 잠재 변수 z가 모델이 다중 모달 시연을 처리할 수 있게 합니다. 액션 청킹이 시간적 일관성을 처리합니다. 이 두 기법이 함께 ACT를 양팔 조작에 매우 효과적으로 만듭니다.

**성능:** 최소한의 데이터로 양팔 조작 벤치마크 (ALOHA 태스크)에서 최첨단. 200개 미만의 시연으로 새 태스크를 학습했습니다.

**단점:**
- 제한된 언어 일반화 (소형 언어 인코더, LLM 없음)
- 인터넷 규모 사전 학습의 혜택을 받지 못함

## Diffusion Policy (디퓨전 정책)

**핵심 아이디어:** CNN 또는 ViT 관찰 인코더와 함께 디퓨전 모델을 액션 헤드로 사용합니다. LLM 없음.

**아키텍처:**
- 관찰 인코더: CNN (ResNet-18) 또는 ViT
- 액션 헤드: DDPM 디퓨전 모델
  - 노이즈 제거 네트워크: 1D 시간적 UNet 또는 MLP
  - 액션 예측: K 스텝 청크 (일반적으로 K=16-32)
  - 디퓨전 스텝: 100 (또는 DDIM으로 16-20)
- 액션 표현: 연속값 (빈닝 없음)

**학습:**
- 데이터셋: teleop 시연; 100개 미만의 시연으로 잘 작동
- 손실: 노이즈 제거 스코어 매칭 (예측된 노이즈에 대한 MSE)

**핵심 인사이트:** 디퓨전이 단일 스텝 MSE 정책이 할 수 없는 다중 모달 액션 분포를 포착합니다. 여러 유효한 파지 구성이나 모호한 접근 방향이 있는 태스크에서 Diffusion Policy는 모드를 평균화하는 대신 유효한 모드로 안정적으로 수렴합니다.

**단점:**
- 언어 일반화 없음 (LLM 또는 대형 언어 인코더 없음)
- DDPM 추론: 100 스텝 × ~1ms = ~100ms/액션; DDIM 없이 너무 느림
- VLM 사전 학습의 혜택을 받지 못함

## π0 (Physical Intelligence)

**핵심 아이디어:** 대형 사전 학습된 VLM 백본 (PaliGemma)과 플로우 매칭 액션 헤드를 결합합니다; 대규모 크로스 구현 데이터로 사전 학습; 태스크별 파인튜닝.

**아키텍처:**
- 백본: PaliGemma (3B VLM: SigLIP + Gemma LLM)
- 융합: 선형 투영을 통해 LLM 공간으로 비전 토큰 투영; 로봇 상태를 선두 토큰으로
- 액션 헤드: K 스텝 액션 청크에 대한 플로우 매칭 (K=50)
- 추론: ~10 스텝으로 플로우 매칭 ODE; 실시간에 충분히 빠름

**학습:**
- 사전 학습: 대형 독점 크로스 구현 로봇 데이터셋 (여러 로봇 유형, 다양한 태스크)
- 파인튜닝: 소량의 데이터셋으로 목표 로봇에 태스크 특화 파인튜닝
- 손실: 플로우 매칭 속도 회귀

**핵심 인사이트:** 대형 VLM 백본(언어 및 시각 일반화)과 플로우 매칭 헤드(정밀하고 다중 모달 액션 생성)의 조합이 언어 일반화와 조작 정밀도를 모두 달성합니다 — 이전 시스템에서 긴장 관계에 있던 두 가지 능력.

**성능:** 2025년 초 기준 많은 로봇 조작 벤치마크에서 현재 최첨단. 새로운 물체와 지시사항으로의 강한 제로샷 일반화를 시연합니다.

**단점:**
- 대형 VLM 백본 (3B)이 파인튜닝과 추론에 상당한 연산이 필요
- 독점 사전 학습 데이터가 재현성 제한
- 파인튜닝 절차가 복잡함 (올바른 학습률, LoRA 설정, 데이터 혼합)

## 비교 표

| 아키텍처 | 백본 | 액션 헤드 | 학습 데이터 규모 | 추론 속도 | 언어 일반화 | 조작 정밀도 |
|---------|------|---------|----------------|---------|-----------|-----------|
| RT-2 | PaLM-E 562B | 이산 토큰 | 매우 큼 (독점) | 매우 느림 (~2s) | 탁월 | 중간 |
| OpenVLA | Llama-2 7B + SigLIP | 이산 토큰 | 큼 (OXE, 공개) | 느림 (~100ms) | 좋음 | 중간 |
| ACT | 소형 트랜스포머 | CVAE 청크 | 작음 (~50-200 시연) | 빠름 (~20ms) | 나쁨 | 탁월 |
| Diffusion Policy | ResNet/ViT | DDPM 디퓨전 | 작음 (~50-100 시연) | 중간 (DDIM ~50ms) | 없음 | 탁월 |
| π0 | PaliGemma 3B | 플로우 매칭 | 매우 큼 (독점) | 중간 (~30ms) | 탁월 | 탁월 |

## 핵심 설계 축

모든 VLA 아키텍처는 네 가지 축 어딘가에 위치합니다:

**축 1: LLM 백본 vs 경량 인코더**
- LLM 백본: 강한 언어 일반화, 비싼 연산, 사전 학습 가중치 필요
- 경량 인코더: 빠름, 태스크 특화, 언어 일반화 없음

**축 2: 토크나이제이션 vs 연속 액션 헤드**
- 토크나이제이션: LLM과 자연스럽게 통합, 제한된 정밀도, 느린 디코딩
- 연속 (MLP/디퓨전/플로우): 더 나은 정밀도, LLM 토큰 예측 불필요

**축 3: 단일 스텝 vs 청킹 예측**
- 단일 스텝: 최대 반응성, 오차 누적 문제
- 청킹: 시간적 일관성, 청크 중간 돌발 상황에 덜 반응

**축 4: 태스크 특화 학습 vs 크로스 구현 사전 학습**
- 태스크 특화: 데이터 요구 사항 적음, 제한된 일반화
- 크로스 구현: 대규모 데이터 인프라 필요, 강한 일반화 보상

2023년 이후 트렌드는 LLM 백본 + 연속 액션 헤드 + 청킹 + 크로스 구현 사전 학습의 조합을 향하고 있습니다 — π0로 예시됩니다.

## 생각해볼 거리

- 새로운 태스크에서 RT-2/OpenVLA와 ACT 중에 선택해야 한다면, 결정 요소는 언어 일반화가 조작 정밀도보다 중요한지입니다 — ACT는 제한된 데이터로 특정 태스크에서 거의 항상 더 정밀하지만, OpenVLA는 ACT가 처리할 수 없는 지시사항 변형을 처리할 수 있습니다.
- 새 VLA를 구축하는데 π0가 목표라면, π0와 이전 시스템 간의 성능 차이는 주로 아키텍처가 아닌 사전 학습 데이터 규모 때문임을 주목하세요 — 태스크 특화 데이터만으로 학습된 π0 스타일 아키텍처는 좁은 벤치마크에서 ACT를 능가하지 못합니다.
- 배포 연산이 하드 제약인 경우 (예: Jetson에서 온로봇 추론), 실용적 아키텍처 공간은 다음으로 줄어듭니다: ACT (빠름, 작음), 양자화된 OpenVLA (중간), 또는 증류/가지치기된 플로우 매칭 헤드 — 3B에서의 전체 π0는 실시간 제어 속도에서 경계선입니다.
- 조작 태스크에 강력한 언어 이해와 밀리미터 수준 정밀도가 모두 필요하다면 (예: 언어 조건화 조립), 현재 어떤 오픈소스 아키텍처도 이것을 완전히 해결하지 못합니다 — PaliGemma와 함께 π0의 플로우 매칭 헤드가 가장 가깝지만, 정밀도 간극을 메우기 위해 여전히 태스크 특화 파인튜닝이 필요할 것입니다.
