# VLA를 위한 딥러닝 — 기초

이 페이지는 VLA 시스템의 근간을 이루는 딥러닝 메커니즘을 다룹니다: 매핑 문제 정의 방식, 각 인코더의 역할, 사용되는 손실 함수, 그리고 실전 최적화 방법.

## VLA 매핑 문제

VLA 정책(policy)은 다음 함수입니다:

```
f(image_t, language, state_t) → action_t
```

매 타임스텝마다 정책은 하나 이상의 카메라 프레임, 자연어 작업 설명, 현재 로봇 고유감각(proprioceptive) 상태를 입력으로 받아 액션 명령을 출력합니다. 이것은 표면적으로 분류나 회귀처럼 보이지만 세 가지 측면에서 더 어렵습니다:

**비정상성(Non-stationarity).** 올바른 액션은 현재 상태에 따라 달라지며, 상태는 이전 액션들의 결과로 변합니다. 학습 분포(시연 데이터)는 특정 상태 궤적을 반영합니다. 배포 시 정책은 오차가 발생하는 즉시 그 궤적에서 벗어나는 상태를 방문하게 됩니다.

**연속적, 고차원 출력.** 6-DOF 엔드이펙터 액션과 그리퍼 상태는 7차원 연속 벡터입니다. 이를 예측하려면 회귀(모드 평균화 위험)나 생성적 디코딩(느린 추론 비용)이 필요합니다.

**다중 모달성(Multi-modality).** 많은 조작 작업에서 여러 액션 시퀀스가 정답일 수 있습니다. 단순 MSE로 학습된 정책은 모드들을 평균화하여 모두에 대해 틀린 액션을 생성합니다. 이것이 디퓨전(diffusion)과 플로우 매칭(flow matching) 액션 헤드의 핵심 동기입니다.

**시간적 의존성.** 정책은 작업 진행 상황을 암묵적으로 추적해야 합니다 (물체를 집었는가? 서랍이 열렸는가?). 단일 프레임, 무상태(stateless) 정책은 이에 어려움을 겪습니다. 액션 청킹(action chunking)과 순환 아키텍처가 부분적 해결책입니다.

## 인코더 역할

### 비전 인코더 (ViT)

Vision Transformer는 입력 이미지를 패치 토큰 시퀀스와 선택적 CLS 토큰으로 인코딩합니다:

```
이미지 (H × W × 3)
  → N개 패치로 분할 (각 패치: patch_size × patch_size × 3)
  → 임베딩 차원 d로 선형 투영
  → 위치 임베딩 추가
  → 트랜스포머 인코더 레이어들
  → N개 패치 토큰 + CLS 토큰 시퀀스
```

224×224 이미지에 패치 크기 16을 사용하면 (224/16)^2 = 196개 패치 토큰이 생성됩니다. CLS 토큰 또는 패치 토큰의 평균 풀링이 전역 이미지 표현으로 사용됩니다. 공간 세부 정보가 액션 헤드에 필요할 때는 전체 패치 시퀀스가 사용됩니다.

**사전 학습 가중치가 중요한 이유.** ImageNet이나 CLIP에서 사전 학습된 ViT는 수백만 개의 이미지에서 물체, 질감, 공간적 관계, 의미론적 범주를 인식하는 방법을 학습했습니다. 로봇 비전을 위해 랜덤 가중치에서 시작하면 유사한 특성 품질에 도달하기 위해 몇 배나 더 많은 데이터가 필요합니다. 사전 학습 인코더는 방대한 로봇 데이터 없이도 시각적 이해를 "무료로" 제공합니다.

### 언어 인코더

텍스트 지시사항은 토크나이징되어 밀집 임베딩(dense embedding) 시퀀스로 인코딩됩니다. LLM 백본 VLA에서는 언어를 처리하는 동일한 트랜스포머가 시각 토큰도 처리합니다. 경량 VLA에서는 고정된 CLIP 텍스트 인코더나 BERT 스타일 모델이 텍스트 임베딩을 별도로 제공합니다.

**언어 임베딩은 작업 조건화 신호 역할을 합니다.** 동일한 물리적 조작을 다르게 설명할 수 있습니다 ("컵을 집어라", "머그잔을 잡아라", "빨간 물체를 가져와라") — 좋은 언어 인코더는 이것들을 거의 동일한 표현으로 매핑해야 합니다. 사전 학습된 언어 모델은 이미 이를 수행하지만, 처음부터 학습하면 그렇지 않습니다.

### 상태 인코더

고유감각 상태(관절 각도, 속도, 엔드이펙터 포즈, 그리퍼 상태)는 저차원 연속 벡터입니다(일반적으로 7-14개 부동소수점). 처리 방식:

1. 학습 데이터셋 통계를 사용해 [-1, 1]로 정규화
2. 소형 MLP로 고정 크기 임베딩으로 인코딩
3. 선두 토큰으로 추가하거나 CLS에 합산하여 백본에 주입

고유감각이 없으면 정책은 카메라만으로 엔드이펙터의 정확한 위치를 추론해야 합니다 — 가림(occlusion) 및 카메라 보정 오류로 인해 이는 매우 데이터 집약적입니다.

## VLA를 위한 손실 함수

손실 함수 선택은 액션 헤드 설계와 밀접하게 연결되어 있습니다.

### MSE / Huber (회귀)

MLP 회귀 헤드에 사용됩니다.

```
L_MSE = (1/T) * sum_t || a_hat_t - a_t ||^2

L_Huber = { 0.5 * (a_hat - a)^2              if |a_hat - a| < delta
           { delta * (|a_hat - a| - 0.5*delta)  그 외
```

Huber 손실은 이상치 시연 액션에 MSE보다 덜 민감합니다. 두 손실 모두 모드 평균화 문제가 있습니다: 주어진 상태에서 두 가지 동등하게 유효한 액션이 있다면 손실 최솟값은 그 사이 — 물리적으로 잘못된 값일 수 있습니다.

### 교차 엔트로피 (이산 토크나이제이션)

액션을 이산 빈(bin)으로 양자화할 때 사용합니다.

```
L_CE = -sum_k a_k * log(p_k)
```

각 액션 차원은 B개 빈에 대한 분류 문제로 취급됩니다. 전체 액션 벡터는 토큰 시퀀스로 자기회귀적으로 예측됩니다. LLM 백본과 자연스럽게 맞아 떨어집니다. 비용: 빈 수에 따른 정밀도 손실, 순차적 디코딩(느림). RT-2, OpenVLA에서 사용됩니다.

### 노이즈 제거 스코어 매칭 (디퓨전)

디퓨전 액션 헤드에 사용됩니다.

```
L_diffusion = E_{t,eps} [ || eps - eps_theta(a_t + sigma_t * eps, sigma_t, obs) ||^2 ]
```

네트워크는 주어진 노이즈 레벨에서 액션에 추가된 노이즈를 예측하는 법을 학습합니다. 추론 시 가우시안 노이즈에서 시작해 반복적으로(일반적으로 10-100 스텝) 깨끗한 액션을 복원합니다. 다중 모달 분포를 포착하지만 추론이 느립니다.

### 플로우 매칭 (Flow Matching)

플로우 매칭 액션 헤드에 사용됩니다 (예: π0).

```
L_flow = E_{t, a_0, a_1} [ || v_theta(a_t, t, obs) - (a_1 - a_0) ||^2 ]
```

a_0 ~ N(0,I), a_1은 목표 액션, a_t = (1-t)*a_0 + t*a_1입니다. 네트워크는 노이즈에서 액션으로의 직선 플로우 속도 필드를 학습합니다. 디퓨전보다 학습이 간단하며 추론 스텝이 적습니다(일반적으로 10개).

## 실전 최적화

### AdamW

VLA 파인튜닝의 표준 옵티마이저입니다. AdamW는 가중치에 직접 가중치 감쇠(weight decay)를 적용하여 사전 학습 모델의 파인튜닝을 안정화합니다.

```
theta_t = theta_{t-1} - lr * m_hat / (sqrt(v_hat) + eps) - lr * lambda * theta_{t-1}
```

lambda는 가중치 감쇠 계수입니다 (VLA 파인튜닝에서 일반적으로 0.01-0.1).

### 학습률 스케줄링

**코사인 감쇠**가 표준입니다:

```
lr_t = lr_min + 0.5 * (lr_max - lr_min) * (1 + cos(pi * t / T))
```

처음 1-5% 스텝 동안 선형 웜업, 이후 작은 하한(lr_min ≈ 1e-6)까지 코사인 감쇠.

**VLA 파인튜닝에 작은 학습률이 필요한 이유.** 백본(LLM 또는 VLM)에 큰 학습률(예: 1e-4)을 적용하면 몇 천 스텝 내에 사전 학습된 특성들을 덮어씁니다 — 이를 **파국적 망각(catastrophic forgetting)**이라고 합니다. VLA 파인튜닝 중 일반적인 백본 학습률: 1e-5 ~ 5e-6. 무작위 초기화된 액션 헤드는 첫 번째 단계에서 더 큰 학습률(1e-4 ~ 1e-3)을 사용할 수 있습니다.

### 그래디언트 클리핑

```
if ||grad|| > max_norm:
    grad = grad * max_norm / ||grad||
```

전역 그래디언트 노름을 클리핑하여 폭발하는 그래디언트를 방지합니다. 일반적인 max_norm: 1.0.

### 혼합 정밀도 학습

순방향 및 역방향 패스에 FP16(또는 BF16)을 사용하고 옵티마이저 업데이트에는 FP32 마스터 가중치를 사용합니다. LLM 백본 VLA에는 BF16이 선호됩니다 — FP32와 동일한 지수 비트를 가져 긴 학습 중 그래디언트 언더플로우 위험이 줄어듭니다.

## 과적합 vs 일반화

### 작은 로봇 데이터셋 문제

로봇 시연 데이터셋은 인터넷 규모 데이터셋에 비해 작습니다. 일반적인 고품질 teleop 데이터셋은 태스크당 100-10,000개의 시연을 포함합니다. VLA 백본은 수백만에서 수십억 개의 파라미터를 가집니다. 정규화 없이는 모델이 시연을 일반화하는 대신 암기합니다.

**완화책:**
- **백본 동결** — 처음에는 액션 헤드만 학습; 백본의 사전 학습 특성이 강한 사전(prior) 역할
- **LoRA 파인튜닝** — 전체 백본 가중치 대신 저랭크 어댑터 행렬(r=8 ~ 64) 학습; 과적합할 파라미터가 훨씬 적음
- **조기 종료** — 보류된 시연에 대한 검증 손실 모니터링; 액션 헤드가 학습셋 아티팩트를 맞추기 전에 중단

### 공동 학습을 정규화 도구로

로봇 데이터와 인터넷 비전-언어 데이터(이미지 캡셔닝, VQA 등)를 학습 중 혼합하면 백본이 일반적인 시각적 특성을 잊지 않습니다. 인터넷 데이터는 백본 표현을 광범위하게 유지하는 제약 역할을 합니다. RT-2는 공동 학습이 새로운 지시사항으로의 일반화를 크게 향상시킨다는 것을 보여주었습니다.

### 데이터 증강

시각 입력: 랜덤 크롭, 색상 지터(hue, saturation, brightness), 가우시안 블러, 랜덤 수평 플립(액션 부호 반전과 함께). 정책이 배경 외양과 액션 사이의 허위 상관관계에 의존하는 것을 방지합니다.

고유감각 입력: 시연 관절 각도에 작은 가우시안 노이즈 추가. 정책이 시연 궤적 근처의 약간의 편차에 강건해지도록 강제합니다.

## 생각해볼 거리

- VLA 학습 손실이 빠르게 수렴하지만 배포 성능이 나쁘다면, 모델이 학습 환경의 배경 외양 단서를 암기한 것입니다 — 더 큰 모델이 아니라 공격적인 이미지 증강과 환경 다양성이 해결책입니다.
- LLM 백본을 로봇 데이터로 파인튜닝할 때 언어 일반화가 저하된다면, 백본 학습률이 너무 높습니다 — 코사인 웜업과 함께 5e-6 이하로 설정하면 거의 항상 언어 이해가 보존됩니다.
- 액션 헤드가 MSE 손실을 사용하는데 로봇이 두 가지 파지 방향 사이에서 망설인다면, 시연 데이터의 다중 모달성이 틀린 중간값으로 평균화되고 있는 것입니다 — 데이터를 더 추가하기 전에 디퓨전 또는 플로우 매칭 헤드로 전환하세요.
