# VLA를 위한 학습 전략

VLA를 어떻게 학습시키느냐는 아키텍처를 어떻게 설계하느냐만큼 중요합니다. 이 페이지는 기본 행동 복제(behavior cloning)에서 VLM 파인튜닝, 공동 학습, 크로스 구현 사전 학습에 이르는 전체 학습 스펙트럼을 다룹니다.

## 행동 복제(Behavior Cloning, BC) 기준선

행동 복제는 가장 간단한 형태의 모방 학습: 시연된 액션을 재현하도록 정책을 학습합니다.

```
데이터셋: D = {(obs_i, action_i)}  인간 시연으로부터 수집

학습 루프:
  for each (obs, a_demo) in D:
    a_pred = policy(obs)
    loss = L(a_pred, a_demo)   # MSE, 교차 엔트로피, 또는 스코어 매칭
    loss.backward()
    optimizer.step()
```

**BC가 주는 것:**
- 시연자의 스타일과 품질을 모방하는 정책
- 학습이 빠름 (지도 학습; RL 보상 불필요)
- 성능이 시연 품질과 다양성에 따라 확장

**BC가 주지 않는 것:**
- 회복 행동 (시연자는 실수를 한 적이 없으므로 정책도 회복 방법을 학습하지 못함)
- 시연 분포 너머의 일반화 (새로운 물체, 배경, 지시사항)
- 최적성 (시연자가 최적 전략을 실행하지 않았을 수 있음)

**BC를 위한 데이터 요구 사항.** 경험적 경험칙으로:
- 10-50 시연: 고정된 장면의 단일 좁은 태스크에 충분
- 100-500 시연: 가벼운 장면 변화에 걸쳐 일반화 (물체 배치, 색상)
- 1,000+ 시연: 여러 물체 유형과 배경에 걸친 강건한 일반화에 필요
- 10,000+ 시연: 다양한 지시사항에 걸친 다중 태스크 일반화에 필요

## 데이터 수집: Teleop vs 스크립트 vs 시뮬레이션

| 방법 | 데이터 품질 | 다양성 | 비용 | 확장성 | Sim2Real 갭 |
|------|-----------|-------|------|-------|------------|
| 인간 원격 조작(teleop) | 높음 (자연스러운 동작) | 높음 | 비쌈 (인간 시간) | 제한적 | 없음 |
| 스크립트 정책 | 중간 (깔끔) | 낮음 (설계된 시나리오) | 쌈 | 높음 | 없음 |
| 시뮬레이션 + 스크립트 | 중간 | 높음 (랜덤화 가능) | 쌈 | 매우 높음 | 있음 (브릿지 필요) |
| 시뮬레이션 + RL | 높음 (최적) | 높음 | 보통 연산 | 매우 높음 | 있음 |
| 반자율 (인간 + 로봇) | 중간-높음 | 중간 | 보통 | 보통 | 없음 |

**프로덕션 VLA를 위한 일반적 데이터 수집 파이프라인:**
1. 목표 환경에서 50-200개 인간 teleop 시연 수집 (시드 데이터)
2. 시드 데이터로 초기 정책 학습
3. 인간 수정/개입과 함께 정책을 실행하여 더 많은 데이터 생성
4. 반복

"개입 루프" (DAgger 또는 그 변형이라고도 함)가 초기 시연 분포 너머로 확장하는 핵심입니다.

## VLM에서 파인튜닝

사전 학습된 VLM (LLaVA, PaliGemma, InstructBLIP 등)에서 시작하는 것이 언어 능력을 가진 VLA를 구축하는 표준 접근 방식입니다.

### 1단계: 동결된 백본, 액션 헤드만 학습

```
학습률 스케줄:
  비전 인코더: 동결됨 (LR = 0)
  LLM 백본: 동결됨 (LR = 0)
  투영 레이어: LR = 1e-4
  액션 헤드: LR = 1e-3

기간: 로봇 데이터에서 10-50k 스텝
목표: 백본을 파괴하지 않고 액션 헤드를 합리적인 출력으로 초기화
```

이 단계는 빠르고 저렴합니다: 소형 액션 헤드만 업데이트됩니다.

### 2단계: 작은 학습률로 전체 파인튜닝

```
학습률 스케줄 (웜업이 있는 코사인):
  비전 인코더: LR = 1e-5 ~ 5e-6
  LLM 백본: LR = 1e-5 ~ 5e-6
  액션 헤드: LR = 1e-4

웜업: 500-2000 스텝
기간: 혼합 데이터 (로봇 + 인터넷)에서 50k-200k 스텝
```

백본의 작은 학습률이 **파국적 망각(catastrophic forgetting)**을 방지합니다. 백본의 학습률이 5e-5를 초과하면, 모델은 일반적으로 몇 천 스텝 내에 언어 일반화의 대부분을 잃습니다.

### LoRA 파인튜닝

모든 백본 가중치를 업데이트하는 대신, 어텐션 레이어에 저랭크 어댑터 행렬을 추가합니다:

```
원래 가중치: W (d_in × d_out)
LoRA 분해: W' = W + A @ B
  A: (d_in × r),  B: (r × d_out),  r << d_in, d_out

A와 B만 학습됩니다. W는 동결됩니다.

일반적인 r: 8 ~ 64
파라미터 감소: d_in*d_out에서 r*(d_in+d_out)으로
  d_in=d_out=4096, r=16에서: 16.7M → 레이어당 131k 파라미터 (128× 감소)
```

LoRA는 다음을 크게 줄입니다:
- 그래디언트 연산에 필요한 메모리 (A와 B만 그래디언트를 받음)
- 파국적 망각 위험 (동결된 W가 사전 학습 특성을 보존)
- 학습 시간 (최적화할 파라미터가 더 적음)

LoRA는 로봇 데이터에서 7B+ 모델을 파인튜닝하는 권장 방법입니다. 단일 태스크 파인튜닝에 r=16-32; 다중 태스크에 r=64를 사용하세요.

## 웹 데이터와 공동 학습(Co-training)

로봇 데이터와 인터넷 비전-언어 데이터를 혼합하면 백본이 파인튜닝 중 일반적인 시각적, 언어적 이해를 잃는 것을 방지합니다.

**메커니즘:**
- 이미지 캡셔닝, VQA, 지시 따르기 예제를 학습 배치에 추가
- 이 예제들은 로봇 액션 목표가 없습니다 — 손실이 언어 출력에만 계산됨
- 인터넷 데이터 손실을 통해 백본 가중치가 사전 학습 값에 가깝게 유지되도록 제약

**데이터 비율.** 경험적으로 로봇 데이터가 학습 배치의 5-20%여야 합니다:
- 너무 낮음: 로봇 태스크 성능이 낮게 유지됨
- 너무 높음: 언어 이해의 파국적 망각

**공동 학습이 새로운 지시사항으로의 일반화를 개선합니다.** RT-2의 핵심 발견: VQA 데이터로 공동 학습된 정책은 "컵과 같은 색의 냅킨 위에 병을 놓아라"와 같은 지시사항을 이 지시사항 표현에 대한 직접적인 학습 예제 없이 실행할 수 있습니다 — VQA 데이터에서 유사한 합성 언어를 보았기 때문입니다.

## 크로스 구현(Cross-Embodiment) 사전 학습

다양한 로봇에서 단일 정책을 학습한 다음 목표 로봇에 파인튜닝합니다.

**데이터셋:**
- **Open X-Embodiment (OXE):** 22가지 로봇 유형, 1M+ 궤적, 다양한 조작 태스크; 공개적으로 사용 가능
- **BridgeData V2:** 24개 환경에서 단일 로봇의 60k+ 시연
- **RT-X:** Google의 대규모 내부 크로스 로봇 데이터셋

**왜 작동하는가.** 형태(morphology) 차이에도 불구하고, 많은 조작 하위 태스크가 공유 구조를 가집니다: 물체 접근, 파지, 배치. 로봇에 걸쳐 학습된 정책은 이러한 공유 표현을 학습합니다. 파인튜닝은 공유된 시각-운동 특성을 보존하면서 로봇 특화 부분(액션 공간, 관찰 특성)을 적응시킵니다.

**핵심 과제: 액션 공간 이질성.** 서로 다른 로봇은 다른 관절 구성, 작업 공간 크기, 제어 인터페이스를 가집니다. 정규화된 좌표로 공통 액션 공간(엔드이펙터 델타)으로 정규화하는 것이 크로스 로봇 학습에 필수적입니다.

**크로스 구현 체크포인트에서 파인튜닝:**
- 위에서 설명한 1단계와 2단계 파인튜닝 사용
- 액션 공간 차원수가 변하면 액션 헤드 재초기화 필요
- 백본 파인튜닝 학습률: 1e-5 이하 (크로스 구현 체크포인트가 순수 VLM 체크포인트보다 더 나은 사전(prior)이므로 더 신중하게 보존)

## 다중 태스크 학습

언어 지시사항에 조건화된 단일 정책으로 여러 태스크를 동시에 학습합니다.

**요구 사항:**
- 데이터셋에서 지시사항 다양성: 각 시연이 자연어 설명으로 레이블됨
- 태스크 균형: 학습 배치에서 각 태스크의 대략 동일한 표현 (그렇지 않으면 정책이 과도하게 특화됨)
- 지시사항 커버리지: 정책이 지시사항 표현에 걸쳐 일반화하도록 패러프레이즈와 동의어 포함

**이점:**
- 태스크 전반의 공유 표현이 일반화 개선
- 태스크가 학습된 시각적 특성을 공유할 수 있으므로 총 데이터 요구 사항 감소
- 많은 태스크를 위한 단일 배포 모델

**과제:**
- 더 어려운 최적화: 태스크들이 충돌하는 그래디언트를 가질 수 있음
- 태스크 간섭: 한 태스크의 개선이 다른 태스크를 저하시킬 수 있음
- 지시사항 특이성: 모호한 지시사항 ("물체를 집어라")이 모호한 학습 신호를 초래

**완화: 태스크 그래디언트 가중치.** 성능이 낮은 태스크에 더 높은 학습 가중치를 부여합니다. 이것은 커리큘럼 학습과 유사하며 다중 태스크 학습 균형을 맞추는 데 도움이 됩니다.

## 데이터 증강

### 이미지 증강

```
RandomCrop:     원본 크기의 90-95%로 크롭 후 다시 리사이즈
ColorJitter:    밝기 (±30%), 대비 (±30%), 채도 (±30%), 색조 (±10%) 랜덤 변동
GaussianBlur:   랜덤 커널 크기로 블러 (20-50% 확률로 적용)
GaussianNoise:  sigma=0.01-0.05로 N(0, sigma) 노이즈 추가
RandomFlip:     수평 플립 (액션 부호를 해당하게 반전할 때만 유효)
```

**중요:** 이미지를 플립하면 액션도 플립해야 합니다 (예: 델타 엔드이펙터 액션의 y 성분 부정). 이것을 하지 않으면 액션-관찰 쌍이 손상됩니다.

### 고유감각 증강

시연된 관절 각도에 작은 가우시안 노이즈 추가:

```
q_aug = q_demo + N(0, sigma_q),  sigma_q ~ 0.01-0.03 라디안
```

이는 정책이 시연 궤적에서의 약간의 편차에 강건하도록 강제합니다 — 실행 오류로 인해 배포 중 직면하는 정확한 분포입니다.

### 카메라 포즈 섭동

고정 카메라의 경우: 시뮬레이션에서 카메라 외부 파라미터(extrinsics)에 작은 랜덤 섭동 추가, 또는 렌더링된 이미지에 원근 왜곡 적용. 정책이 세션에 걸친 작은 카메라 정렬 불량에 불변하도록 학습시킵니다.

## 생각해볼 거리

- VLA 파인튜닝 실행이 빠르게 떨어지는 학습 손실을 보이지만 검증 손실이 평탄하다면, 더 나은 아키텍처가 필요한 것이 아닙니다 — 더 다양한 시연 데이터가 필요합니다; 모델이 장면 특화 외양 단서를 암기하고 있지 태스크 관련 행동을 학습하지 않고 있습니다.
- 웹 VQA 데이터와의 공동 학습이 명백한 일반화 이점 없이 학습을 느리게 한다면, 로봇 데이터 비율이 너무 낮은 것입니다 (5% 미만) — 백본이 로봇 관련 특성을 인코딩할 기회가 없습니다; 조작 검증 지표가 개선되기 시작할 때까지 로봇 데이터 비율을 높이세요.
- r=8로 LoRA 파인튜닝을 시도했는데 전체 파인튜닝보다 낮은 성능을 보인다면, 즉시 랭크를 높이지 마세요 — 먼저 액션 헤드 (LoRA가 아닌 완전히 파인튜닝되어야 함)가 충분한 용량을 가지는지 확인하세요; LoRA 랭크는 대부분의 조작 태스크에서 완전한 액션 헤드 업데이트보다 덜 중요합니다.
