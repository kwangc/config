# 액션 공간 설계

액션 공간(action space)은 정책이 매 타임스텝에 출력하는 것을 정의합니다. 이것은 근본적인 설계 결정입니다: 어떤 행동이 표현 가능한지를 제약하고, 정책이 로봇 형태(morphology)에 걸쳐 얼마나 일반화되는지를 결정하며, 학습 안정성에 영향을 미칩니다.

## 액션이란 무엇인가?

액션은 정책이 매 타임스텝에 생성하는 출력 벡터로, 로봇 하드웨어에 명령으로 전송됩니다(직접 또는 중간 컨트롤러를 통해). 액션은 다음이 될 수 있습니다:

- **저수준**: 관절 토크 또는 속도; 빠른 제어 루프 필요 (1 kHz); VLA에서 거의 사용 안 함
- **관절 각도 목표값**: 원하는 관절 위치; 하위 컨트롤러(PD/PID)가 실행 처리
- **엔드이펙터 포즈 목표값**: 원하는 데카르트 포즈 (위치 + 방향); IK 솔버가 관절 목표값으로 변환
- **고수준 웨이포인트(waypoints)**: 계획된 궤적의 경유점; 모션 플래너가 실행
- **의미적 액션**: "파지", "놓기", "밀기"; 실행할 프리미티브 라이브러리 필요

대부분의 조작 VLA는 5-50 Hz에서 **관절 각도** 또는 **엔드이펙터 포즈** 수준에서 동작합니다.

## 관절 공간(Joint Space)

직접 관절 각도 목표값: 정책이 라디안 단위의 원하는 관절 위치를 출력합니다.

```
action = [q1_target, q2_target, ..., q7_target, gripper_target]
```

| 특성 | 세부 사항 |
|------|----------|
| 차원수 | 팔당 7 (회전 6 + 그리퍼 1) |
| 전이성(Transferability) | 낮음 — 로봇 형태에 특화 |
| 정밀도 | 높음 — 직접 하드웨어 명령 |
| 해석 가능성 | 낮음 — 작업 공간에서 추론 어려움 |
| 제어 주파수 | VLA에서 일반적으로 10-50 Hz |

**사용 시점:** 단일 로봇 플랫폼을 가지고 크로스 구현(cross-embodiment) 전이가 필요 없을 때; 관절 수준에서 밀리미터 이하 정밀도가 필요할 때; IK 런타임 비용을 피하고 싶을 때.

**피해야 할 시점:** 다중 로봇 학습이나 서로 다른 팔 모델에 걸친 데이터셋 — 관절 각도 범위와 의미가 로봇 계열에 따라 다릅니다.

## 엔드이펙터(태스크) 공간

정책이 로봇 엔드이펙터의 6-DOF 포즈를 출력합니다: 위치 (x, y, z)와 방향(오일러 각도 또는 쿼터니언).

```
action = [dx, dy, dz, rx, ry, rz, gripper]   # 델타 엔드이펙터
     or = [x, y, z, qx, qy, qz, qw, gripper]  # 절대값 엔드이펙터
```

| 특성 | 세부 사항 |
|------|----------|
| 차원수 | 6-7 DOF + 그리퍼 |
| 전이성 | 중간-높음 — 위치/방향은 로봇 독립적 |
| 정밀도 | 중간 — IK 정확도에 따라 다름 |
| 해석 가능성 | 높음 — 작업 기하학에 직접 매핑 |
| 제어 주파수 | VLA에서 일반적으로 10-30 Hz |

**태스크 공간이 더 잘 전이되는 이유.** 5 cm 앞으로 이동하는 것은 어떤 팔 모델이 실행하든 동일합니다. 관절 공간 명령은 팔 특화 운동학을 인코딩하지만, 엔드이펙터 명령은 작업 기하학을 인코딩합니다. 크로스 구현 데이터셋(OXE 등)은 일반적으로 로봇에 걸쳐 학습을 가능하게 하기 위해 엔드이펙터 공간으로 정규화합니다.

## 델타(Delta) vs 절대값(Absolute) 액션

### 델타 (상대적) 액션

정책이 현재 상태에서의 변화를 출력합니다:

```
delta_action = [Δx, Δy, Δz, Δrx, Δry, Δrz, gripper_state]
new_pose = current_pose + delta_action
```

**장점:**
- 자연스럽게 일반화 가능: "2 cm 앞으로 이동"은 팔이 현재 어디 있든 동일한 행동
- 더 작은 출력 범위 (작은 델타), 학습하기 쉬움
- 오류가 지역적: 한 스텝의 나쁜 델타가 팔을 멀리 떨어진 잘못된 위치로 점프시키지 않음

**단점:**
- 장기 목표는 많은 누적 스텝이 필요; 드리프트가 누적됨
- 먼 목표를 효율적으로 지정하기 어려움

### 절대값 액션

정책이 목표 포즈를 직접 출력합니다:

```
absolute_action = [x_target, y_target, z_target, rx_target, ry_target, rz_target, gripper]
```

**장점:**
- 장거리 이동에 효율적: 한 스텝으로 큰 이동 명령 가능
- 해석하기 쉬움: 목표가 특정 월드 프레임 위치

**단점:**
- 정책이 로봇의 작업 공간 경계를 알아야 함
- 잘못된 절대값 목표는 팔을 잠재적으로 위험한 위치로 보냄
- 다른 시작 구성에 걸쳐 일반화하기 어려움

**일반적 관행:** 접촉이 풍부한 조작 태스크에는 **델타 엔드이펙터 액션**을 사용하세요. 웨이포인트 기반 모션 플래너와 함께 작업하거나 대략적인 위치 지정 단계에는 **절대값 액션**을 사용하세요.

## 그리퍼 상태

그리퍼 상태는 일반적으로 팔 액션과 함께 별도 출력으로 처리됩니다:

| 표현 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 이진 (0/1) | 열림 또는 닫힘 | 간단, 모호함 없음 | 파지 힘 제어 불가 |
| 연속값 [0,1] | 부분적 파지 폭 | 다양한 물체에 표현력 있음 | 더 정밀한 감독 필요 |
| 이산 토큰 | 빈으로 구분된 파지 위치 | LLM 토크나이제이션에 맞음 | 정밀도 손실 |

이진 그리퍼는 대부분의 피킹과 플레이싱 태스크에 충분합니다. 연속값 파지는 섬세한 물체(과일, 폼) 또는 특정 파지 폭이 필요한 태스크(펜 파지, 세밀한 조립)에 중요해집니다.

## 양팔(Bimanual) 액션 공간

양팔 로봇(예: ALOHA, 이팔 설정)의 경우 액션 공간이 두 배가 됩니다:

```
bimanual_action = [왼팔 (7D), 왼쪽 그리퍼 (1D), 오른팔 (7D), 오른쪽 그리퍼 (1D)]
               = 16D 합계
```

### 협조 제약 조건

양팔 액션은 독립적이지 않습니다: 두 팔이 동일한 작업 공간을 공유하므로 충돌하지 않아야 합니다. 각 팔의 액션을 독립적으로 출력하는 정책(양팔을 두 개의 병렬 단팔 정책으로 취급)은 물리적으로 호환되지 않는 액션을 생성합니다.

협조를 강화하는 방법:
- **양팔 시연에 대한 공동 학습**: 정책이 인간 teleop 데이터에서 협조된 행동을 학습; 명시적 제약 없이 데이터 분포가 암묵적으로 실현 가능성을 강화
- **팔 간 공유 어텐션**: 왼팔과 오른팔 토큰 사이에 어텐션이 있는 트랜스포머 기반 액션 헤드; 명시적 팔 간 상호작용 가능
- **순차적 예측**: 왼팔을 먼저 예측하고, 왼팔에 조건화하여 오른팔 예측; 자기회귀적 협조

ACT는 K=100의 액션 청킹(action chunking)을 통해 강건한 양팔 조작을 시연한 최초의 아키텍처 중 하나였습니다 — 단일 청크 내에서 두 팔의 일관된 궤적을 계획할 수 있었기 때문입니다.

## 정규화된 액션 공간

모든 액션 차원은 학습 전에 정규화되어야 합니다. 정규화 없이는:

- 큰 원시값을 가진 차원 (예: 밀리미터 단위의 x 위치: 0-500 mm)이 손실을 지배
- 작은 원시값을 가진 차원 (예: 라디안 단위의 관절 각도: 0-3 rad)이 과소적합
- 액션 헤드의 MLP나 디퓨전 모델이 이질적인 스케일에 걸쳐 효율적으로 학습 불가

**표준 정규화:**

```
# 방법 1: 데이터셋 통계를 사용해 [-1, 1]로 정규화
a_norm = 2 * (a - a_min) / (a_max - a_min) - 1

# 방법 2: z-점수 정규화
a_norm = (a - a_mean) / a_std
```

이상치 처리를 위해 정규화 후 [-1, 1]로 클립합니다. 추론 시 역변환을 적용해 물리적 단위를 복원합니다.

**차원별 별도 정규화**: 각 액션 차원은 자체 정규화 통계를 가집니다. 모든 차원을 공동으로 정규화하지 마세요.

## 액션 공간 차원수와 정책 복잡도

| 액션 표현 | 차원수 | 전이성 | 학습 난이도 |
|----------|-------|-------|------------|
| 관절 토크 | 팔당 6-7 | 매우 낮음 | 매우 높음 |
| 관절 각도 | 팔당 6-7 | 낮음 | 높음 |
| 델타 EEF (오일러) | 7 (xyz + rpy + grip) | 중간 | 중간 |
| 델타 EEF (쿼터니언) | 8 (xyz + quat + grip) | 중간 | 중간-높음 (단위 제약) |
| 양팔 델타 EEF | 14-16 | 중간 | 높음 |
| 청킹됨 (K 스텝) | K × 7 | 중간 | 높음 (긴 시퀀스) |

더 높은 차원수는 정책이 더 많은 출력 차원에 걸쳐 동시에 일관된 값을 생성해야 하기 때문에 학습을 더 어렵게 만듭니다. 액션 청킹(action chunking)은 유효 출력 차원을 7에서 K×7(예: K=100에서 700)로 확장합니다. 이것이 청킹 아키텍처가 단순한 MLP 회귀 대신 표현력 있는 액션 헤드(ACT의 CVAE, π0의 디퓨전)가 필요한 이유입니다.

## 생각해볼 거리

- 단일 로봇 정책이 재학습 후 동일한 유형의 두 번째 로봇으로 전이에 실패한다면, 액션 공간 정규화 통계가 새 로봇 데이터에서 재계산되었는지 확인하세요 — 첫 번째 로봇의 오래된 정규화는 모델이 그 외에는 동일하더라도 이동된 출력을 생성합니다.
- 양팔 정책을 설계하면서 MLP에서 왼팔과 오른팔 출력을 단순히 쌓는다면, 정책은 두 팔을 독립적으로 취급합니다 — 동시 접촉이 필요한 태스크에서의 첫 번째 실패가 이것을 드러낼 것입니다; 처음부터 두 팔 토큰에 걸쳐 공유 트랜스포머를 사용하세요.
- 정책이 시뮬레이션에서 부드러운 궤적을 생성하지만 물리 로봇에서 저키한 명령을 생성한다면, 델타 액션이 더 적절한 곳에 절대값 액션을 사용하고 있는 것입니다 — 각 예측된 절대값 목표는 새로운 점프이지 작은 부드러운 증분이 아닙니다.
- 그리퍼 상태 출력이 파지 중에 열림과 닫힘 사이에서 진동한다면, 연속 그리퍼 출력이 시간적 일관성 제약 없이 학습되고 있는 것입니다 — 그리퍼를 이산화하거나 액션 헤드에 시간적 스무딩 제약을 추가하세요.
