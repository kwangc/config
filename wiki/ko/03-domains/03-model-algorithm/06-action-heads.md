# 액션 헤드 아키텍처

액션 헤드(action head)는 VLA 파이프라인의 최종 모듈입니다. 백본의 출력 표현(벡터 또는 토큰 시퀀스)을 받아 액션을 생성합니다. 이것은 사소한 설계 선택이 아닙니다: 전체 VLA의 표현력, 학습 역학, 추론 속도가 액션 헤드에 크게 의존합니다.

## 액션 헤드란?

백본(트랜스포머, LLM, 또는 CNN)이 시각적 + 언어적 + 상태 입력을 처리한 후, 표현을 생성합니다 — 일반적으로 차원 d의 벡터 (예: 7B LLM에서 d=2048). 액션 헤드는 이 표현을 액션으로 변환합니다:

```
backbone_representation: (batch, d)
              |
       [액션 헤드]
              |
     action: (batch, action_dim)   (예: 델타 EEF에서 (batch, 7))
```

액션 헤드는 VLA 파인튜닝을 위해 사전 학습된 백본 위에 추가되는 유일한 새 구성요소입니다. 헤드 설계는 다음을 결정합니다:

- 다중 모달 액션 분포를 포착할 수 있는지
- 추론이 얼마나 빠른지
- 학습을 위한 감독 신호가 얼마나 풍부한지
- 헤드가 백본의 학습 목표와 호환되는지

## MLP 회귀 헤드

가장 간단한 액션 헤드: 하나 이상의 완전 연결 레이어가 백본 임베딩을 액션 벡터로 매핑합니다.

```
backbone_emb: (d,)
   → Linear(d, hidden_dim) + ReLU
   → Linear(hidden_dim, hidden_dim) + ReLU
   → Linear(hidden_dim, action_dim)
   = action_pred: (action_dim,)
```

**손실:** 예측된 액션과 시연 액션 사이의 MSE 또는 Huber 손실.

**장점:**
- 매우 빠른 추론 (<1 ms)
- 구현 및 디버그 간단
- 시연 데이터가 낮은 다중 모달성을 가질 때 잘 작동 (상태당 하나의 올바른 액션)

**단점 — 모드 평균화.** 주어진 상태에 두 가지 동등하게 유효한 액션이 존재하면 (예: 왼쪽 또는 오른쪽에서 물체 파지), MSE 최소화는 평균을 생성합니다 — 두 경우 모두 틀린 액션. 이 실패 모드는 여러 유효한 전략이 있는 조작 태스크에서 일반적입니다.

**사용처:** RT-1, 초기 조작 정책, 소거 연구의 기준선 헤드.

## 이산 토크나이제이션 (자기회귀)

연속 액션을 이산 값으로 빈닝하고, 액션 시퀀스를 일련의 토큰으로 자기회귀적으로 예측합니다.

```
범위 [a_min_i, a_max_i]와 B개 빈을 가진 액션 차원 i에 대해:
  bin = floor((a - a_min_i) / (a_max_i - a_min_i) * B)

토큰 시퀀스로서의 액션: [bin_1, bin_2, ..., bin_D]
  (D = action_dim, 예: 델타 EEF에서 7개 토큰)

예측: 토큰당 B개 클래스에 대한 소프트맥스, 한 번에 하나씩 예측
손실: 각 토큰 위치에서 교차 엔트로피
```

**LLM 백본에 맞는 이유.** LLM은 이미 어휘에 대한 토큰 로짓을 생성합니다. 액션 토큰을 어휘에 추가하고 자기회귀적으로 예측하는 것은 아키텍처 변경이 필요 없습니다 — 액션은 시퀀스 끝의 더 많은 텍스트 토큰일 뿐입니다.

**장점:**
- LLM 파이프라인에 자연스럽게 맞음
- 교차 엔트로피 손실이 MSE보다 더 풍부한 학습 신호 제공
- 토큰 분포를 통해 일부 불확실성 표현 가능

**단점:**
- **정밀도 손실**: B=256 빈과 10 cm 범위로 각 빈은 ~0.4 mm — 수용 가능하지만 빈 수가 정확도 대 어휘 크기에 영향
- **순차적 디코딩**: D개 토큰이 한 번에 하나씩 디코딩; 각 디코딩 스텝이 LLM을 통한 전체 순방향 패스; D=7이고 7B 모델에서 단일 순방향 패스보다 7× 느림
- **여전히 모드 평균화**: 토큰 분포 아래 기댓값은 주어진 관찰에 대해 여전히 모드들의 평균

**사용처:** RT-2, OpenVLA.

## 디퓨전 헤드 (Diffusion Head)

액션은 역 디퓨전(reverse diffusion) 과정으로 생성됩니다: 가우시안 노이즈에서 시작해 학습된 노이즈 제거 네트워크를 반복적으로 적용하여 깨끗한 액션을 복원합니다.

```
순방향 과정 (학습, 추론 시 불필요):
  a_noisy = a_clean + sigma_t * epsilon,  epsilon ~ N(0, I)

역방향 과정 (추론):
  a_T ~ N(0, I)
  for t = T, T-1, ..., 1:
    epsilon_pred = eps_theta(a_t, t, observation)
    a_{t-1} = a_t - step_size * epsilon_pred   (단순화된 DDPM 업데이트)
  최종 액션 = a_0

학습 손실 (노이즈 제거 스코어 매칭):
  L = E [ || epsilon - eps_theta(a_t, t, obs) ||^2 ]
```

노이즈 제거 네트워크 eps_theta는 일반적으로 관찰 표현에 조건화된 소형 MLP 또는 1D UNet입니다.

**디퓨전이 다중 모달 분포를 포착하는 이유.** 각 노이즈 레벨에서 노이즈 제거 네트워크는 노이즈가 있는 액션을 데이터 분포 쪽으로 어느 방향으로 밀어야 하는지를 학습합니다. 두 개의 모드(두 가지 유효한 파지)가 있는 분포가 자연스럽게 처리됩니다: 네트워크는 가장 가까운 모드 쪽으로 밀도록 학습하며, 추론 시 다른 랜덤 시드가 다른 유효한 모드를 생성합니다.

**장점:**
- 복잡한 다중 모달 액션 분포 포착
- 모드를 평균화하지 않음 — 각 추론 샘플이 일관된 유효한 액션
- 유연한 노이즈 제거 네트워크가 어떤 아키텍처도 될 수 있음

**단점:**
- **느린 추론**: T 노이즈 제거 스텝 × 순방향 패스 비용; T=100이 표준 DDPM; DDIM으로 T=10-20
- **학습 복잡성**: 노이즈 스케줄, 디퓨전 스텝 수, 네트워크 아키텍처 선택에 신중한 튜닝 필요
- **LLM 자기회귀 디코딩과 기본적으로 호환되지 않음**

**추론 가속 기술:**
- DDIM (Denoising Diffusion Implicit Models): 재학습 없이 스텝을 100에서 10-20으로 감소
- 일관성 모델: 원스텝 생성; 일부 품질 손실
- 몇 스텝 증류: 학생 모델이 더 적은 스텝으로 좋은 액션을 생성하도록 학습

**사용처:** Diffusion Policy, Octo (디퓨전 변형), π0.

## 플로우 매칭 헤드 (Flow Matching Head)

플로우 매칭(flow matching)은 단순한 노이즈 분포에서 액션 분포로의 연속 정규화 플로우를 학습하는 것으로 액션 생성을 프레이밍합니다.

```
데이터: a_0 ~ p_data (목표 액션), a_noise ~ N(0, I)
보간: a_t = (1-t) * a_noise + t * a_0,  t in [0, 1]
목표 속도: v_t = a_0 - a_noise  (t에 걸쳐 일정, 즉 직선 경로)

학습 손실:
  L = E_t [ || v_theta(a_t, t, obs) - v_t ||^2 ]

추론 (ODE 솔버, 예: 오일러):
  a_0 = a_noise
  for t = 0 to 1 with step dt:
    a += v_theta(a, t, obs) * dt
  최종 액션 = a
```

**디퓨전에 비한 핵심 장점:**
- **더 간단한 학습 목표**: 속도 필드로의 단일 회귀 (노이즈 스케줄이 있는 스코어 함수가 아님)
- **더 직선적인 궤적**: 노이즈에서 데이터로의 경로가 (기댓값으로) 대략 직선이므로 ODE 솔버 스텝이 더 적게 필요
- **더 빠른 추론**: 일반적으로 10 스텝으로 충분; 때로는 스텝 크기 튜닝으로 1-3 스텝

**실용적 고려 사항:**
- 여전히 MLP 회귀보다 느림 (10 스텝 vs 1 스텝)
- 디퓨전처럼 다중 모달 분포를 포착하기에 충분히 표현력 있음
- 학습 단순성으로 인해 대부분의 새로운 VLA 아키텍처에서 디퓨전보다 선호됨

**사용처:** π0 (Physical Intelligence), 최근 여러 연구 VLA.

## 비교

| 헤드 종류 | 표현력 | 추론 속도 | 학습 복잡성 | 모드 포착 | 적합한 용도 |
|----------|-------|----------|------------|---------|-----------|
| MLP 회귀 | 낮음 | 매우 빠름 (<1 ms) | 낮음 | 아니오 | 단순 단일 모달 태스크; 소거 기준선 |
| 이산 토크나이제이션 | 중간 | 느림 (순차 디코딩) | 낮음 (CE 손실) | 부분적 | LLM 네이티브 파이프라인; 언어 풍부 태스크 |
| 디퓨전 (DDPM) | 매우 높음 | 느림 (50-100 스텝) | 높음 | 예 | 다중 모달 조작; 연구 벤치마크 |
| 디퓨전 (DDIM 몇 스텝) | 높음 | 중간 (10-20 스텝) | 높음 (추론 시 빠름) | 예 | 디퓨전 이점을 가진 실용적 배포 |
| 플로우 매칭 | 매우 높음 | 중간 (5-10 스텝) | 중간 | 예 | 새 VLA; 최선의 속도-표현력 트레이드오프 |

## 헤드 선택 방법

**MLP를 선택할 때:**
- 태스크가 상태당 하나의 주요 올바른 액션을 가질 때
- 추론 레이턴시가 중요할 때 (<2 ms)
- 파이프라인의 나머지를 검증하기 위한 빠른 기준선이 필요할 때

**이산 토크나이제이션을 선택할 때:**
- LLM 백본 위에 구축하고 아키텍처 단순성을 원할 때
- 언어 일반화가 조작 정밀도보다 중요할 때
- 느린 디코딩을 허용하거나 액션 청킹으로 분할 상환 가능할 때

**디퓨전을 선택할 때:**
- 태스크에 진정한 다중 모달성이 있을 때 (여러 유효한 파지, 접근 각도)
- 조작 정밀도가 중요할 때
- 추론 시 10-20 노이즈 제거 스텝을 감당할 수 있을 때

**플로우 매칭을 선택할 때:**
- 더 간단한 학습과 더 빠른 추론으로 디퓨전 수준의 표현력을 원할 때
- 새 VLA 아키텍처를 처음부터 시작할 때
- π0 또는 π0에서 영감을 받은 파이프라인을 사용할 때

## 생각해볼 거리

- 간단한 피킹과 플레이싱 태스크에서 디퓨전 헤드와 MLP 헤드를 벤치마크하여 유사하게 수행한다면, MLP가 충분하다고 결론짓지 마세요 — 여러 유효한 파지 방향이 있는 태스크를 추가하면 MLP는 즉시 모드 평균화 실패를 드러내고 디퓨전은 그렇지 않습니다.
- 디퓨전 헤드의 추론이 실시간 제어에 너무 느리다면, 완전히 다른 헤드를 고려하기 전에 10 스텝의 DDIM으로 전환하세요 — 100에서 10 스텝으로의 품질 손실은 보통 작으며, 10× 레이턴시 감소가 종종 디퓨전을 실시간 예산 내로 가져옵니다.
- 대규모 크로스 구현 데이터 사전 학습 후 새로운 태스크에 플로우 매칭 헤드를 파인튜닝한다면, 1,000 그래디언트 스텝 내에 일반적으로 안정적입니다 — 수렴하지 않는다면 헤드 아키텍처가 아니라 액션 정규화 통계가 잘못된 것입니다.
- B=256 빈의 이산 토크나이제이션을 사용하는데 로봇이 상수 오프셋으로 목표를 지나친다면, 빈 경계가 시연된 액션의 실제 범위 중앙에 있지 않습니다 — 이상치 처리를 위해 최솟값/최댓값 대신 데이터셋 분위수에서 빈 경계를 재계산하세요.
