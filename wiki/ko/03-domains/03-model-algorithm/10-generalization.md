# 일반화와 강건성

실험실에서는 잘 작동하지만 현장에서 실패하는 VLA는 연구 아티팩트가 아닙니다 — 일반화를 의도적으로 설계하지 않으면 기본 결과입니다. 이 페이지는 VLA가 직면하는 분포 이동(distribution shift)의 분류 체계, 실패 방식, 그리고 해결 방법을 다룹니다.

## 분포 이동 분류 체계

VLA 정책은 관찰, 지시사항, 로봇 상태에 대한 특정 분포를 가진 데이터셋으로 학습됩니다. 배포 시 이 학습 분포와 배포 환경 사이의 불일치가 분포 이동입니다. 네 가지 유형이 있습니다:

| 이동 유형 | 변하는 것 | 일반적 실패 | 감지 방법 |
|---------|---------|-----------|---------|
| 시각적 이동 | 조명, 배경, 물체 외양 | 정책이 물체를 무시하거나 잘못 식별 | 학습과 배포 간 이미지 통계 비교 |
| 물리적 이동 | 물체 질량, 표면 마찰, 관절 마모 | 액션이 물리적으로 올바르지만 힘이 불충분 | 힘/토크 센서 이상값 |
| 태스크 이동 | 새로운 지시사항 표현, 새로운 태스크 조합 | 정책이 지시사항을 잘못 해석하거나 무시 | 언어 임베딩 거리 |
| 구현 이동 | 다른 로봇, 다른 카메라 장착 | 완전히 잘못된 액션 | 해당 없음 (아키텍처 수준) |

**VLA 정책은 조용히 실패합니다.** 낮은 신뢰도를 출력할 수 있는 분류기와 달리, 회귀 또는 디퓨전 액션 헤드는 항상 액션을 생성합니다. 정책은 완전히 잘못되었을 때도 자신감 있게 실행합니다. 이것이 근본적인 강건성 과제입니다: 모델은 자신이 무엇을 모르는지 모릅니다.

## 시각적 분포 이동

배포 시 가장 일반적인 실패 모드입니다. ViT 인코더는 학습 세트의 조명 조건, 배경 외양, 물체 질감에서 특성을 학습했습니다. 새로운 환경 — 다른 주변 조명, 다른 테이블 표면, 새로운 물체 색상 변형 — 이 패치 토큰 분포를 충분히 이동시켜 정책이 잘못된 액션을 생성합니다.

**왜 실험실 평가 중에 보이지 않는가.** 학습과 동일한 물리적 설정에서 평가하면 시각적 이동이 없습니다. 정책이 훌륭하게 보일 수 있습니다. 고객 시설에 배포하면 (다른 조명, 다른 테이블, 다른 주변 물체) 성능이 붕괴됩니다.

### 완화책

**다양한 학습 환경.** 가장 신뢰할 수 있는 완화책입니다. 학습 데이터가 많은 다른 조명 조건, 배경, 물체 색상 변형을 커버하면 정책이 이것들에 불변하도록 학습됩니다.

**도메인 랜덤화.** 시뮬레이션에서: 학습 중 조명 색상, 강도, 배경 질감, 물체 색상, 카메라 위치를 랜덤화합니다. 실제 데이터에서: 여러 방, 시간대, 다양한 배경에서 시연을 수집합니다.

**학습 중 이미지 증강.** 랜덤 색상 지터, 가우시안 노이즈, 랜덤 크롭이 ViT를 작은 시각적 섭동에 불변하도록 학습시킵니다. 진정한 장면 다양성을 대체하지는 않지만 조명 변화에 대한 민감도를 줄입니다.

**손목 카메라를 불변 시점으로.** 손목 카메라는 전역 장면보다 로봇 자신의 상태와 높은 상관관계가 있는 근거리 시점을 제공합니다. 배경 변화에 고정 오버헤드 카메라보다 더 불변합니다.

## 물리적 분포 이동

로봇의 역학이 변합니다. 모터가 노화되거나, 물체가 예상보다 무겁거나, 표면이 더 미끄럽거나, 그리퍼 패드가 마모됩니다. 명목 역학으로 작동하도록 학습된 정책의 액션 명령이 더 이상 효과적이지 않습니다.

**완화 전략:**

- **라이브 통계로 고유감각 정규화.** 관절 위치 오프셋을 설명하기 위해 주기적으로 proprio 정규화를 업데이트합니다.
- **힘/토크 감지를 입력으로.** 손목 F/T 센서 읽기값을 관찰에 추가합니다. 정책이 접촉 피드백을 기반으로 파지 힘이나 접근 각도를 조정하는 방법을 학습할 수 있습니다.
- **잔류 정책.** 기본 정책의 출력에 대한 수정을 예측하는 소형 잔류 정책을 학습합니다. 기본 정책이 명목 행동을 제공하고 잔류가 적응합니다.
- **루프 내 시스템 식별.** 각 태스크 전에 간단한 ID 궤적을 실행하여 현재 시스템 역학을 추정하고 정책을 추정된 파라미터에 조건화합니다.

## 배포 시 오차 누적

분포 이동 없이도 결정론적 정책은 긴 궤적에 걸쳐 작은 오차를 누적합니다. K 스텝 후의 상태는 K개의 불완전한 액션이 순차적으로 적용된 결과입니다; 의도한 궤적에서의 편차가 증가합니다.

**오차 누적 속도는 다음에 따라 달라집니다:**
- 태스크 길이 (더 많은 스텝 = 더 많은 오차 누적)
- 정책 오차 크기 (작은 오차 × 많은 스텝 = 큰 편차)
- 상태 피드백 빈도 (액션 청킹이 피드백을 줄여 청크당 오차를 증가시킬 수 있음)

**완화:**

- **액션 청킹 + 시간적 앙상블:** [07 — 액션 청킹](./07-action-chunking/) 참조
- **회복 행동:** 실패 상태와 회복 시연에서 명시적으로 학습; 정책이 실패를 감지하고 재계획하는 방법을 학습
- **폐루프 접촉 제어:** 접촉이 풍부한 단계 (삽입, 핀인홀)의 경우, VLA에서 더 높은 주파수의 반응 컨트롤러(임피던스 제어, 힘 제어)로 핸드오프

## 분포 외(OOD) 감지

VLA가 조용히 실패하기 때문에, 명시적 OOD 감지가 안전한 배포에 중요합니다.

### 액션 출력에 대한 신뢰도 점수

디퓨전 또는 플로우 매칭 헤드의 경우: N개의 추론 샘플을 실행하고 예측된 액션의 분산을 계산합니다. 높은 분산은 모델이 올바른 액션에 대해 불확실하다는 신호입니다.

```
actions = [sample_action(policy, obs) for _ in range(N)]
uncertainty = std(actions)   # 차원별 또는 집계
if uncertainty > threshold:
    trigger_recovery()
```

이산 토크나이제이션 헤드의 경우: 액션 토큰 로짓에 대한 소프트맥스 엔트로피. 낮은 엔트로피 = 자신감; 높은 엔트로피 = 불확실.

### 앙상블 불일치

N개의 정책(다른 랜덤 시드 또는 다른 데이터 서브셋)을 학습하고 각 스텝에서 불일치를 측정합니다:

```
disagreement = std([policy_i(obs) for i in range(N)])
```

높은 불일치 = OOD 상태. 비싸지만 (N× 추론 비용) 신뢰할 수 있습니다.

### 휴리스틱 경계

배포에 간단하고 실용적:
- **속도 경계:** 예측된 액션이 안전 한계를 초과하는 관절 속도를 명령하면 잘못된 것일 가능성이 높음
- **작업 공간 경계:** 예측된 엔드이펙터 목표가 작업 공간 외부라면 거부하고 홀드
- **액션 부드러움:** 예측된 액션이 이전 액션에서 delta_max 이상 점프하면 클립 또는 거부

이것들이 모든 실패를 감지하지는 않지만 가장 위험한 것들을 포착합니다.

## 평가 vs 일반화

일반적인 실수: VLA 성능을 학습 설정에서만 평가하고 높은 성공률을 보고합니다. 이것은 암기와 일반화를 혼동합니다.

**좋은 평가에 포함되어야 하는 것:**

| 평가 유형 | 테스트하는 것 | 예시 |
|---------|-----------|------|
| 인-분포 | 암기/보간 | 학습에서 본 위치의 물체 |
| 새로운 물체 위치 | 공간 일반화 | 동일 작업 공간의 새로운 위치에 있는 물체 |
| 새로운 물체 인스턴스 | 시각 일반화 | 동일 카테고리, 다른 외양 (새로운 컵 색상) |
| 새로운 물체 카테고리 | 의미 일반화 | 학습에서 전혀 보지 못한 물체 |
| 새로운 조명 | 시각 불변성 | 다른 주변 조명 아래 평가 |
| 새로운 지시사항 표현 | 언어 일반화 | "머그잔을 잡아라" vs "컵을 집어라" |
| 장기 지평선 | 시간적 강건성 | 순차적 하위 태스크가 필요한 10+ 스텝 태스크 |

**제로샷 vs 퓨샷 일반화.** 제로샷: 정책이 어떤 파인튜닝 없이 새로운 조건에서 평가됩니다. 퓨샷: 1-10개의 새 시연이 평가 전에 제공됩니다. 대부분의 프로덕션 배포는 배포 환경에서 어느 정도의 파인튜닝을 사용합니다.

**일반화 지표.** 동시에 적용되는 분포 이동의 수의 함수로 성공률을 보고합니다. 하나의 이동(새 조명)을 처리하지만 두 개의 동시 이동(새 조명 + 새 물체)에서 실패하는 정책은 강건성 한계에 대해 정보를 제공합니다.

## 핵심 레버로서의 데이터 다양성

일반화의 가장 신뢰할 수 있는 단일 예측 변수는 학습 데이터 다양성입니다. 학습에서 더 많은 고유한 장면, 물체 인스턴스, 조명 조건, 지시사항 표현, 시연자 스타일 → 더 나은 일반화.

**다양성을 위한 데이터 수집 전략:**
- 처음부터 여러 방/조명 조건에서 시연 수집
- 카테고리당 다양한 물체 사용 (머그잔 하나가 아닌 10-20개)
- 여러 시연자 포함 (다른 동작 스타일이 정책을 정규화)
- 의도적으로 물체를 비정상적인 위치와 각도에 배치
- 작업 공간 어수선함 랜덤화 (장면의 관련 없는 물체)

**하드웨어-인-더-루프 다양성.** 가장 데이터 효율적인 다양성 전략: 부분적으로 학습된 정책을 배포하고, 태스크를 시도하게 하고, 실패를 수집하고, 실패 회복 시연을 추가합니다. 정책 자신의 실패 분포가 정확히 학습되어야 할 분포입니다.

## 생각해볼 거리

- 정책이 실험실에서 90%+ 성공을 달성하지만 고객 현장에서 50% 미만이라면, 간극은 거의 항상 시각적 분포 이동입니다 — 고객 데이터로 재학습하기 전에, 조명 차이의 영향을 줄이는 진단으로 이미지 전처리에 밝기/화이트 밸런스 정규화 단계를 추가하세요.
- 새로운 VLA 아키텍처를 벤치마크할 계획이라면, 인-분포 조건 외에 새로운 물체 인스턴스와 새로운 지시사항 표현에 대한 평가를 실행하세요 — 새로운 조건에서 사라지는 인-분포 벤치마크의 5포인트 개선은 더 잘 일반화하는 것이 아니라 더 잘 암기하는 아키텍처입니다.
- 정책이 실험실 테스트에서 파지 실패로부터 잘 회복하지만 프로덕션에서는 그렇지 않다면, 학습의 회복 시연이 기본 태스크와 동일한 장면 설정에서 수집되었습니다 — 회복 행동은 기본 태스크만큼 많은 장면 변형에 걸쳐 학습되어야 합니다.
- 앙상블 불일치가 보류된 테스트 세트에서 실패를 올바르게 예측한다면, 모델은 이미 내부적으로 일부 상태가 불확실하다는 것을 "알고" 있습니다 — 이는 학습 데이터 분포가 이미 경계를 포착하고 있음을 의미하며, 그 경계 근처에 더 다양한 데이터를 추가하면 어떤 아키텍처 변경보다 더 빠르게 성능이 개선됩니다.
