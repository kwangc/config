# ML Fundamentals (신경망 & Transformer)

---

## 핵심 개념 (채워나가기)

- **신경망(NN)**은 데이터를 통해 학습되는 함수 근사기다. (가중치 + 학습 목표가 핵심)
- **학습**은 반복 루프다: 순전파 → Loss → 역전파 → Optimizer 업데이트.
- **Transformer**는 attention 기반 시퀀스 모델로, 현대 LLM/VLM/VLA 스택의 기반이 된다.

---

## 신경망: 무엇을 하는가

- **레이어**는 입력을 표현으로 바꾸고, 연결된 **가중치**가 그 변환 규칙을 결정한다.
- **활성화 함수**는 비선형성을 추가해서 복잡한 함수를 표현할 수 있게 한다.
- **Loss(목표 함수)**는 예측과 정답의 차이를 측정하며, 이를 최소화하도록 학습한다.

## 학습 루프: loss → gradient → 업데이트

- **데이터셋 분리**: train은 학습, validation은 튜닝, test는 최종 확인.
- **학습률(learning rate)**은 업데이트 크기를 제어한다:
  - 너무 크면: 학습이 불안정/진동
  - 너무 작으면: 학습이 너무 느림
- **일반화**는 “실제로 원하는 것”에 맞는 목표 함수를 얼마나 잘 설계했는지에 달려 있다(로보틱스에서는 특히 중요).

## Transformer & Attention 기초

### 왜 attention인가

- RNN처럼 토큰을 순서대로만 압축하는 대신, Transformer는 각 토큰이 다른 토큰을 직접 참고한다.
- 결과적으로 고정된 순차 상태(state)보다 “문맥에 맞는 정보 혼합”을 만들 수 있다.

### Q · K · V (고수준)

- **Q(Query)**: 이 토큰이 찾는 것(무엇을 원하나)
- **K(Key)**: 각 토큰이 제공하는 것(무엇을 담고 있나)
- **V(Value)**: 실제로 모아서 합칠 정보

### Softmax와 스케일링

- 유사도 점수는 softmax로 정규화된 가중치(확률)로 바뀐다.
- 스케일링은 attention 로그잇이 과도하게 커져 softmax가 한쪽으로만 몰리는 것을 방지한다.

### Multi-head + context window

- **Multi-head attention**은 서로 다른 관계 패턴을 병렬로 학습한다.
- **Context window**는 한 번에 attention할 수 있는 토큰 상한으로, 성능과 비용을 함께 좌우한다.

---

## RNN vs Transformer (실전 포인트)

- RNN은 recurrent hidden state로 정보를 압축하므로 장거리 의존이 희석될 수 있다.
- Transformer는 토큰 간 직접 상호작용이어서 장거리 추론과 GPU 스케일링에 유리하다.

## Food for Thought

- 로보틱스 “성공”은 단순 token 단위 정확도가 아니다. 모델이 행동에 필요한 표현을 학습하도록 목표/평가 신호를 language metric 중심에서 action quality로 옮겨야 할 수 있다.
- attention 기반 모델은 강력하지만 distribution shift(새 환경/새 로봇/새 툴체인)에서 깨질 수 있다. 실패를 먼저 정의하고 다루면, 모델 주변의 인터페이스(스키마/제약)를 더 단단하게 만들 기회가 된다.
- 큰 기회는 attention/학습의 기본 메커니즘을 제품 제약(지연, 결정성, 안전 게이팅)과 연결하는 것이다. 그래야 “스케일링 이론”이 “배포 엔지니어링”으로 이어진다.
