# Glossary

로봇·VLA·데이터 인프라 관련 **핵심 용어** 한 줄 정의 모음.

---

## 로봇·제어

| 용어 | 정의 |
|------|------|
| **Bimanual** | 두 손(듀얼 암/엔드이펙터) 협응으로 수행하는 조작 |
| **Teleoperation (teleop)** | 사람이 원격으로 로봇을 조작해 데모를 수집하는 방식 |
| **Proprioception** | 로봇 자신의 관절·위치·속도 등 상태 감각 |
| **Manipulation** | 그랩, 플레이스, 조립 등 물체를 다루는 행위 |

---

## 시뮬레이션·전이

| 용어 | 정의 |
|------|------|
| **Sim2Real** | 시뮬레이션에서 학습한 정책/모델을 실세계 로봇에 전이하는 것 |
| **Reality gap / Domain gap** | 시뮬과 현실 간 차이(물리·센서·외관 등)로 인한 성능 저하 |
| **Domain randomization** | 시뮬 파라미터(광원, 질감, 마찰 등)를 랜덤화해 전이 성능을 높이는 기법 |

---

## 모델·학습

| 용어 | 정의 |
|------|------|
| **VLA** | Vision-Language-Action — 비전·언어·액션을 한 모델로 다루는 로봇 정책 |
| **Foundation model** | 넓은 태스크/데이터로 사전학습한 뒤 특정 태스크로 파인튜닝하는 모델 |
| **Closing the loop** | 학습 → 실세계 테스트 → 데이터/모델 개선을 반복해 검증하는 과정 |

---

## 배포·인프라

| 용어 | 정의 |
|------|------|
| **On-device deployment** | 모델을 로봇/엣지 디바이스에 올려 클라우드 없이 추론하는 방식 |
| **Quantization** | 가중치/활성화를 저정밀도(예: 4bit, 8bit)로 줄여 속도·메모리 절약 |

---

## 모델·지각·행동

| 용어 | 정의 |
|------|------|
| **VLM** | Vision-Language Model로 입력 비전을 텍스트/구조화 신호로 접지(grounding)하는 모델 |
| **VLA** | Vision-Language-Action — 비전·언어·로봇 상태를 묶어 행동 정책을 만드는 모델 |
| **Action representation (7DoF)** | 7차원(자세/행동) 신호로 행동을 표현해 다운스트림 제어를 더 견고하게 만드는 방식 |
| **Trajectory consistency** | 생성/계획된 행동이 시간 흐름에서 “흔들리지” 않도록 정합성을 유지(종종 DTW로 측정) |

---

## 데이터·스케일·유저빌리티

| 용어 | 정의 |
|------|------|
| **Scenario-based coverage planning** | 임베딩/클러스터링(예: UMAP + HDBSCAN)으로 다음 수집 시나리오를 결정하는 접근 |
| **Data lineage / versioning** | 데이터 버전, 라벨, 스키마, 전처리까지 추적해 재현·비교 가능하게 만드는 체계 |
| **Label noise** | 인간/라벨링이 만드는 실수로, 품질 게이트가 없으면 학습을 멈추게 할 수 있음 |
| **Schema consistency** | 운영자/툴/모델 버전 간 입력·출력 계약을 안정적으로 유지하는 것 |
| **Calibration drift** | 센서→로봇 좌표 정렬이 어긋나 학습 라벨이 깨지는 현상 |
| **Synchronization** | 멀티모달 센서 스트림을 동일 시점으로 맞춰(타임스탬프 정렬) cues가 같은 “물리 순간”을 설명하게 하는 것 |

---

## 학습·적응·운영

| 용어 | 정의 |
|------|------|
| **Behavior cloning (BC)** | 데모로부터 지도학습(이밋)하는 모방학습(종종 워밍업으로 사용) |
| **Reinforcement learning (RL)** | 보상 기반 최적화(예: PPO)로 순수 모방을 넘어 성능을 끌어올리는 방식 |
| **Closed-loop refinement** | 온라인 롤아웃에서 실패를 수집→재학습하는 human-in-the-loop 개선 루프 |
| **Recovery loop** | 실패 후 제어를 회복하기 위한 연산/절차(운영자가 구조화해서 데이터로 남김) |
| **Human-operated gripper** | 목표 로봇에 맞춘 행동 라벨을 추정하기 위해 사람이 조종/운영하는 엔드이펙터 |

---

## 평가·안전 신호

| 용어 | 정의 |
|------|------|
| **DTW (Dynamic Time Warping)** | 시간 차이를 허용하면서도 궤적 일관성을 측정하는 지표 |
| **E-stop** | 안전 제약이 위반되면 즉시 모션을 중단하는 비상 정지 |
| **Torque limits** | 액추에이터가 넘지 말아야 하는 토크 한계값(배포/테스트 안전 경계) |
| **Rollout modes (shadow/canary)** | 단계적 롤아웃 패턴으로 실제 현장에서 리스크를 제한하면서 개선을 검증 |
