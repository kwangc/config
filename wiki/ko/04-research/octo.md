## 2) 상세 내용

### 사전학습 VLA에서 실제 배포까지

Octo는 새로운 로봇에 쉽게 적응할 수 있도록 설계된 완전히 오픈소스의 범용 로봇 정책입니다. 대규모 사전학습 모델(RT-2, OpenVLA)과 제한된 하드웨어를 가진 연구자들의 실제 사용성 간의 간극을 해소합니다.

**철학:** 기본적으로 잘 작동하지만, 새로운 로봇에서의 빠른 파인튜닝을 위해 의도적으로 설계된 강력한 오픈소스 파운데이션을 제공합니다.

### 아키텍처: Transformer + 확산 액션 헤드

- **관찰 토큰화:** 이미지는 패치로(ViT 스타일), 언어 목표는 텍스트 토큰으로 토큰화 후 공유 Transformer로 입력
- **Readout 토큰:** 액션 예측에 최종 표현을 사용하는 특수 학습 가능한 토큰 (모든 토큰이 아닌 readout 토큰만 → 계산 효율)
- **액션 헤드:** **확산 기반** — 노이징 제거로 액션 분포 출력 (Diffusion Policy와 유사), RT-2/OpenVLA처럼 이산 토큰 아님
  - 다중 모달 액션 분포에 더욱 표현력 있음
  - 토큰 기반보다 매끄러운 궤적
- **모듈식 설계:** 관찰 입력(카메라 추가/제거), 액션 표현(로봇 DoF) 교환 용이

### 학습 전략: OXE 파운데이션 + 효율적 파인튜닝

- **OXE에서 사전학습:** 9개 로봇 에구디먼트 80만+ 궤적
- **두 모델 크기:** Octo-Small (27M), Octo-Base (93M 파라미터)
- **완전 공개:** 가중치, 코드, 학습 데이터, 학습 레시피 모두 공개

### 새로운 로봇에 대한 파인튜닝

**핵심 강점:** 빠른 적응을 위해 설계. 실증적으로:
- **50-100개 시연**만으로도 새 로봇에서 처음부터 학습한 정책과 동급 이상
- **500-1000개 시연**으로는 처음부터 학습한 것을 큰 폭으로 초과
- 새로운 관찰 타입 지원 (다양한 카메라 각도, 손목 카메라 등)

### 결과

| 태스크 / 셋업 | Octo-Base | RT-2-X (55B) | 전문가 베이스라인 |
|-------------|-----------|-------------|-------------------|
| BridgeV2 (제로샷) | 56.9% | 67.3% | ~75%+ |
| 파인튜닝 (WidowX, 500개) | **78%** | 평가 안됨 | ~75% |
| 파인튜닝 (UR5, 500개) | 경쟁력 | 평가 안됨 | 베이스라인 |

### 한계

- 제로샷 성능이 RT-2-X, π0보다 뒤짐
- 확산 액션 헤드는 ~10 스텝 필요 (DDPM의 100 스텝보다는 빠르지만 토큰 예측보다는 느림)
- ACT, π0 대비 정밀 / 접촉 많은 조작에 대한 평가 제한적
- 태스크 레벨 목표 수준의 언어 이해만 가능

## 3) 왜 중요한 논문인가

- **패러다임 전환: 사전학습 → 사전학습 + 효율적 파인튜닝:** RT-2, OpenVLA는 제로샷 중심이었다면, Octo는 로봇 파운데이션 모델의 목표가 '적응성'임을 입증
- **오픈소스 범용 정책의 실용성:** Octo의 아키텍처(모듈식, 효율적 readout, 확산 헤드)는 학계 클러스터에서도 파인튜닝 가능하도록 의도적으로 설계됨
- **확산 액션 헤드의 주류화:** 이산 토큰 대신 확산 기반 액션 생성이 실행 가능하고 더 부드러운 궤적을 생성함을 증명
- **오픈소스 로봇 정책의 금 표준 확립:** 완전한 학습 레시피와 함께 Octo는 가장 재현 가능하고 적응하기 쉬운 범용 정책

## 4) Config 적용 사례

- **파인튜닝을 주요 경로로:** Config의 모델 전략은 완벽한 제로샷을 목표하지 말고, 강력한 파운데이션(OXE/OpenVLA/Octo) + Config 특화 양팔 데이터로 적극적 파인튜닝. 이게 처음부터 만드는 것보다 낫습니다
- **파인튜닝된 Octo를 기준으로:** Octo-Base를 500-1k WidowX 시연으로 파인튜닝한 것이 외부 베이스라인; Config는 양팔 정밀 조작에서 이를 초과해야 합니다
- **양팔 조작을 위한 확산 액션 헤드:** Octo의 확산 기반 액션 생성은 높은 자유도, 접촉 많은 양팔 조작에 토큰 기반보다 적합; Config도 확산/흐름 기반 액션 헤드를 채택해야
- **하드웨어 변화에 대응하는 모듈식 아키텍처:** Octo의 설계(카메라 추가/제거 용이, 액션 차원 변경)는 Config의 로봇 형태가 진화해도 정책이 우아하게 적응하도록 보장
- **데이터 포맷 일치:** Octo는 RLDS를 사용하고 OXE로 학습됨; Config의 데이터 파이프라인도 RLDS 호환성을 유지해서 Octo가 실용적 베이스라인이 되도록
