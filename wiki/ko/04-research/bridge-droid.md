## 2) 상세 내용

### 두 가지 보완 가능한 로봇 데이터 전략

BridgeData V2와 DROID는 로봇 시연 데이터를 확장하는 두 가지 보완적 전략을 나타냅니다:

- **BridgeData V2:** 깊이 (단일 로봇, 다양한 태스크, ~24 환경)
- **DROID:** 너비 (여러 로봇, 많은 환경, "현실의" 다양성)

### BridgeData V2: 주방 벤치마크

**규모:** WidowX 로봇으로 수집한 ~60,000개 궤적, 주방형 환경

**구성:**
- WidowX 250 로봇팔 (표준화된 하드웨어)
- 주방형 테이블탑 셋업 (~24개 다양한 주방 연구실)
- 태스크: 집기, 붓기, 서랍 조작, 접기, 풀기 등
- 언어 주석: 각 궤적마다 자연어 태스크 설명 포함
- 완전 오픈소스, 다운로드 가능

### DROID: 현실의 다양성

**규모:** Franka Panda 로봇으로 다양한 환경에서 수집한 ~76,000개 궤적

**구성:**
- **86+ 다양한 현실 환경:** 집, 사무실, 연구실, 카페, 창고
- **50+ 기관 데이터 기여:** 표준화된 수집 프로토콜
- **Franka Panda (동일 그리퍼):** 하드웨어 일관성, 환경 가변성
- 언어 주석: BridgeData V2와 유사
- 가정, 사무, 산업 조작 태스크 포함

**핵심 혁신:** "현실의" 데이터 수집은 현실 환경 다양성(조명, 혼란, 물체 다양성, 배치)이 실험실 큐레이션 데이터보다 더 견고한 정책을 만든다는 것을 입증했습니다.

### 비교: 깊이 vs 너비

| 차원 | BridgeData V2 | DROID |
|----|----|----|
| 규모 | ~60k 궤적 | ~76k 궤적 |
| 로봇 | WidowX (단일) | Franka Panda (단일) |
| 환경 | ~24 실험실 | 86+ 현실 위치 |
| 주요 용도 | 벤치마크 평가 | 사전학습 + 평가 |
| 학습 초점 | 태스크 일반화 | 환경 일반화 |

### 평가 결과

**BridgeData V2 벤치마크:**
- OpenVLA (7B): 73.7% 성공률
- RT-2-X (55B): 67.3% 성공률
- Octo-Base: 56.9% 성공률 (제로샷, 파인튜닝으로 극적 향상)

**DROID 사전학습 효과:**
- DROID에서 사전학습 후 하위 태스크에서 파인튜닝한 모델이 처음부터 학습한 모델 초과
- 제로샷 DROID 평가는 단일 연구실 학습 정책보다 환경 견고성 우수

### 한계

**BridgeData V2:**
- 단일 로봇 타입은 형태 다양성 제한
- 실험실 환경은 현실의 혼란과 조명 변화를 포착하지 못함
- ~60k 영상은 이제 "최소" 수준 — 많은 논문이 더 큰 데이터셋을 기대

**DROID:**
- Franka 전용은 다른 형태로의 일반화 제한
- 성공의 원인을 데이터 품질 vs 환경 다양성으로 명확히 귀인하기 어려움
- 50+ 기관 협력으로 라벨링과 데이터 품질에 약간의 불일치 도입

## 3) 왜 이 데이터셋이 중요한가

- **BridgeV2가 공개 평가 금 표준 확립:** 공개 벤치마크 없이는 모델 비교가 거의 불가능했습니다. BridgeV2의 주방 태스크는 로봇 학습 논문의 기준점이 되었습니다
- **DROID가 우선순위를 규모에서 다양성으로 전환:** 환경 다양성(현실 집 vs 실험실)이 태스크 다양성만큼 중요함을 입증; "더 큰 데이터셋이 항상 낫다" 가정에 도전
- **두 데이터셋이 함께 두 가지 확장 방향 검증:** 깊이(고정 하드웨어의 다양한 태스크)와 너비(고정 하드웨어의 다양한 환경), 둘 다 일반화 향상
- **VLA 시대 가능하게 함:** OpenVLA, Octo, π0은 모두 BridgeData V2에서 벤치마킹하고 OXE(DROID 같은 다양성 집계)에서 학습. 이 공개 데이터셋 없이는 전체 운동이 멈췄을 것

## 4) Config 적용 사례

- **벤치마크 목표:** BridgeV2 성공률(OpenVLA 73.7%, RT-2-X 67.3% 주방 태스크)이 핵심 외부 기준. Config는 양팔 정밀 조작에서 이를 초과해야
- **태스크 vs 환경 다양성 트레이드오프:** DROID의 환경 다양성이 태스크 다양성만큼 중요하다는 발견은 Config가 Config 시설 하나가 아닌 다양한 실제 주방/공장/연구실에서 수집해야 함을 시사
- **공개 벤치마킹:** Config가 양팔 정밀 조작 벤치마크(BridgeV2와 평행)를 구축하면 외부 비교와 커뮤니티 기여가 가능해집니다
- **언어 주석 규율:** BridgeData V2와 DROID 모두 일관된 언어 주석(궤적당 한 문장)을 유지합니다. Config도 동일한 엄격함을 시행해야 합니다
- **규모 있는 데이터 수집:** DROID의 다기관 프로토콜(50+ 파트너 기여)은 내부 오버헤드 없이 데이터를 확장하는 템플릿입니다
