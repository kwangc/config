## 2) 상세 내용

### 로봇 학습의 마일스톤: 멀티-에구디먼트 데이터

Open X-Embodiment (OXE)는 **22개의 서로 다른 로봇 에구디먼트** 에서 **21개 연구 기관** 이 수집한 로봇 시연 데이터를 모은 획기적인 데이터셋입니다. 핵심 통찰: 다양한 인터넷 텍스트와 이미지가 범용적인 언어모델과 비전모델을 만들었듯이, 다양한 로봇 시연 데이터도 더 잘 일반화하는 정책을 만들 수 있다는 것입니다.

이는 Carnegie Mellon, Stanford, Berkeley, MIT, Google DeepMind 등 수많은 기관이 공통 데이터 포맷에 동의하고 자사의 로봇 데이터를 공개하기로 한 대규모 협력의 결과입니다.

### 데이터셋의 규모와 구성

OXE 데이터셋은 다음을 포함합니다:

- **~100만 개의 궤적** — 22개 로봇 에구디먼트
- **527개의 서로 다른 조작 스킬** (집기, 누르기, 밀기, 붓기, 삽입, 열기 등)
- **다중 센서 모달리티:** RGB 카메라 (손목 + 외부), 깊이 센서, 고유수용감각 상태 (관절각, 엔드-이펙터 포즈)
- **혼합 데이터 소스:** 원격조작된 시연, 스크립팅된 정책, 인간 비디오, 다양한 환경
- **모두 공개 릴리스** — 완전히 재현 가능한 학습

### RT-X 학습: OXE 혼합으로 학습된 모델

이 논문은 또한 Robotics Transformer (RT-1, RT-2)를 OXE 전체 크로스-에구디먼트 혼합으로 재학습한 **RT-X** 모델을 소개합니다:

- **RT-1-X:** RT-1 백본을 OXE로 재학습 → 제로샷 일반화 대폭 향상
- **RT-2-X (55B):** RT-2 백본을 OXE로 재학습 → 새로운 에구디먼트와 태스크에서 최강 성능

**핵심 발견:** 다양한 크로스-에구디먼트 데이터로 학습한 모델이 **단일 에구디먼트 데이터로 학습한 모델보다 뛰어납니다** — 원래 에구디먼트에서의 평가에서도 마찬가지입니다.

### 결과

| 벤치마크 | 모델 | 데이터 출처 | 새로운 로봇 제로샷 | 원래 로봇 |
|---------|------|------------|--------------------|-|
| BridgeV2 | RT-2 (단일) | WidowX만 | 낮음 | 67% |
| BridgeV2 | RT-2-X | 전체 OXE | **최고** | **최고** |
| RoboMimic (Franka) | RT-1-X | OXE | 우수 | 유지 |
| RoboMimic (UR5) | RT-1-X | OXE | 경쟁력 | 경쟁력 |

### 한계

- 모든 궤적에 명시적 오픈-어휘 언어 컨디셔닝 없음
- 제로샷 성능은 여전히 새로운 분포의 태스크에서 인간 전문가 수준 미달
- 전체 OXE 혼합으로 모든 모델을 재학습하는 것은 계산 비용이 높음
- 에구디먼트 불균형: 일부 로봇은 더 많은 데이터를 가짐

## 3) 왜 중요한 논문인가

- **로봇 학습의 ImageNet 모멘트:** 멀티-에구디먼트에서 확장하면 일반화가 향상된다는 것을 처음 증명 — 이후 모든 VLA 연구(Octo, OpenVLA, π0)가 OXE를 기반으로 함
- **공개 인프라 돌파:** 21개 기관의 협력이라는 거대한 문제를 해결 — 이 단일 데이터셋이 한 세대의 재현 가능한 로봇 학습 연구를 가능하게 함
- **에구디먼트 전문화 신화 타파:** 다양성이 전문화보다 우수하다는 것을 증명; 다양하게 학습한 모델이 전문가 모델을 이김
- **로봇 정책의 스케일링 법칙:** 신경망 스케일링 법칙(더 큰 모델 + 더 다양한 데이터 = 더 나은 일반화)이 로봇 정책에도 언어/비전만큼 적용됨을 입증

## 4) Config 적용 사례

- **파운데이션 → 전문화 경로:** Config의 모델 개발은 OXE의 레시피를 따르되, 다양한 크로스-에구디먼트 데이터로 먼저 학습한 후 Config 특화 양팔 조작으로 파인튜닝해야 합니다. 이것이 Config만의 데이터로 시작하는 것보다 낫습니다
- **벤치마크 베이스라인:** BridgeV2 평가 태스크가 RT-2-X와 RT-1-X의 표준 비교점입니다. Config도 이 숫자들(>65% 성공률)을 넘어야 합니다
- **데이터 포맷 / RLDS 호환성:** OXE는 RLDS (TFRecord 기반) 형식을 사용합니다. Config가 RLDS 출력을 유지하면 OpenVLA, Octo 같은 OXE 학습 모델을 바로 파인튜닝할 수 있습니다
- **수량보다 다양성:** 기초 학습에서는 다양성이 확장보다 중요합니다; Config는 같은 태스크 반복보다 주방, 실험실, 조립, 정밀 조작 등 여러 유형에서 수집해야 합니다
- **다기관 데이터 협력:** OXE의 성공은 개방 데이터로 가능했습니다. Config도 학계 연구실, 로봇 팀과 데이터 공유를 고려해 전체 수집 부담 없이 다양성을 확장할 수 있습니다
